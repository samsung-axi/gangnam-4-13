[development_analysis.skills 필드 정의]
각 기술에 대해 반드시 다음 필드를 포함한다:
- `name`: 기술명을 명확하게 작성
  예: "엎드린 자세에서 머리 들어올리기", "혼자 앉기", "잡고 서기", "독립 보행" 등
- `category`: "대근육운동", "소근육운동", "인지", "언어", "사회정서" 중 하나
- `present`: true/false (관찰되었는지 여부)
- `frequency`: 관찰된 횟수 (숫자)
- `level`: "없음", "초기", "중간", "숙련" 중 하나
  - 없음: 해당 행동을 전혀 시도하지 않음
  - 초기: 시도는 하지만 대부분 실패하거나 매우 불안정함 (성공률 0-30%)
  - 중간: 때때로 성공하지만 여전히 불안정하거나 도움이 필요함 (성공률 30-70%)
  - 숙련: 안정적이고 반복적으로 성공함, 자신감 있게 수행 (성공률 70% 이상)
- `examples`: 관찰된 구체적인 장면 설명 배열 (최소 1개 이상)
  예: ["00:01:23 - 엎드린 자세에서 3초간 머리를 들고 유지함"]


[development_score 필드 정의 - 매우 중요!]
발달 점수는 아이의 나이대에 맞는 발달 수준을 평가하는 0-100점 척도입니다.

**기본 점수: 70점**
- 나이대에 딱 맞는 표준 발달 수준

**점수 조정 규칙:**
1. **나이대 대비 평가 (stage_consistency 기반)**
   - `match_level: "전형적"` → **점수 유지 (70점)**
     - 해당 월령의 표준 발달 수준에 부합
   
   - `match_level: "약간빠름"` → **+10~+15점 (80-85점)**
     - 다음 단계 수준의 행동이 관찰됨
     - next_stage_signs가 명확함
   
   - `match_level: "약간느림"` → **-10~-20점 (50-60점)**
     - 이 단계의 기대 행동이 거의 관찰되지 않거나 매우 불안정함
     - 숙련도가 전반적으로 "초기" 수준
   
   - `match_level: "많이다름"` (빠르거나 느린 경우):
     - **매우 빠름** → **+20~+30점 (90-100점)**
       - 2단계 이상 앞선 행동 관찰
     - **매우 느림** → **-20~-30점 (40-50점)**
       - 이전 단계 수준에도 미치지 못함

2. **숙련도 평가 (개별 skills의 level 기반)**
   - "숙련" 비율이 높을수록 → **+5~+10점**
   - "초기"나 "없음" 비율이 높을수록 → **-5~-10점**

3. **종합 판단**
   - 영상에서 관찰된 전반적인 발달 수준을 종합적으로 고려하여 최종 점수를 결정하세요.
   - 점수는 반드시 0~100점 사이의 정수여야 합니다.

**예시:**
- 12개월 아이가 혼자 걷기 (전형적) → 70점
- 12개월 아이가 달리기 시도 (약간빠름) → 85점
- 12개월 아이가 잡고 서기도 불안정 (약간느림) → 55점
- 9개월 아이가 능숙하게 혼자 걷기 (많이다름-빠름) → 92점

[development_radar_scores 필드 정의 - 매우 중요!]
영역별 발달 점수를 0-100점으로 평가합니다. **각 영역은 독립적으로 평가**되며, 관찰된 행동에 따라 점수가 달라져야 합니다.

**5개 영역:**
1. **운동** (대근육/소근육 운동 발달)
2. **언어** (언어 이해 및 표현)
3. **인지** (인지 능력 및 문제 해결)
4. **사회성** (타인과의 상호작용)
5. **정서** (정서 표현 및 조절)

**점수 계산 방법:**

1. **기준 점수: 70점** (각 영역)
   - 나이대 기준으로 전형적인 발달 수준 (또래 평균)

2. **관찰된 긍정적 행동에 따라 가점 (최대 100점):**
   - **빈도/일관성:** 나이대 기대 행동이 매우 일관되고 자주 관찰되면 **+5점에서 +15점**.
   - **숙련도:** 행동의 숙련도가 높게 관찰되면 (**'숙련' 수준**) **+5점에서 +15점**.
   - **선행 발달:** 다음 발달 단계의 행동이 관찰되면 **+10점에서 +20점**.
   - (선택적) **월등한 발달:** 2단계 이상 앞선 발달 행동이 관찰되면 **+20점에서 +30점**.
   - 최종 점수는 100점을 초과할 수 없습니다.

3. **관찰된 부정적 행동/기대 행동 미관찰에 따라 감점:**
   - **기대 행동 미관찰 (기본):** 단순 미관찰은 감점하지 않고 **기본 점수(70점)**를 유지하세요. (영상 길이 문제일 가능성 높음)
   - **기대 행동 미관찰 (예외 - 냉철한 판단):** 단, 해당 단계에서 **필수적으로 빈번하게 나타나야 하는 행동**(예: 걸음마 단계의 걷기, 모방 단계의 눈맞춤)이 **충분한 활동/상호작용 상황**임에도 전혀 관찰되지 않는다면 냉철하게 감점하세요. (**-10점에서 -20점**)
   - **퇴행/비전형적 행동:** 나이대에 비정상적이거나 발달 퇴행으로 보이는 행동이 명확히 관찰될 때만 감점하세요. (**-10점에서 -25점**)
   - 최종 점수는 40점 미만으로 내려가지 않도록 합니다.

4. **영역별 평가 원칙:**
   - 각 영역은 독립적으로 평가하되, **억지로 점수 차이를 벌리지 마세요.**
   - 특정 영역(예: 언어, 정서)의 행동이 영상에 찍히지 않았다면 편안하게 **기본 점수(70점)**를 부여하세요. 억지로 점수를 50점대로 깎으면 부모가 불필요하게 걱정합니다.
   - 영상에서 확실히 잘하는 행동이 포착된 영역에만 가점을 부여하여 차별화하세요.

**예시 시나리오:**

**시나리오 1: 12개월 아이, 활발한 운동 활동만 찍힌 경우**
- timeline_observations에 걷기, 기기, 서기 행동이 20회 이상 관찰됨
- 언어, 인지, 사회성/정서 행동은 영상에 특별히 잡히지 않음

→ 결과: (70점 기준 적용)
```json
{
  "운동": 95,  // 매우 활발함 (70 + 15(빈도) + 10(숙련) = 95)
  "언어": 70,  // 관찰되지 않음 → 기본 점수 유지
  "인지": 70,  // 관찰되지 않음 → 기본 점수 유지
  "사회성": 70,  // 관찰되지 않음 → 기본 점수 유지
  "정서": 70   // 관찰되지 않음 → 기본 점수 유지
}
```

**시나리오 2: 24개월 아이, 언어 발달 활발**
- 단어 발화 15회, 2단어 조합 5회 관찰됨
- 다른 영역은 평범하거나 화면에 안 잡힘

→ 결과: (70점 기준 적용)
```json
{
  "운동": 70,  // 기본 수준
  "언어": 95,  // 매우 활발함 (70 + 25)
  "인지": 75,  // 약간의 탐색 관찰
  "사회성": 85,  // 상호작용 활발
  "정서": 70   // 평이함
}
```

**주의사항:**
1. **관찰되지 않은 영역은 70점(평균)으로 둡니다.** (불확실할 땐 평균 점수가 정답입니다)
2. **명확히 관찰된 긍정/부정 행동에 대해서만 가점/감점을 적용하세요.**
3. **억지로 모르는 영역의 점수를 깎아 점수 분포를 넓히지 마세요.**

[stage_consistency 필드 정의]
- `match_level`: "전형적" | "약간빠름" | "약간느림" | "많이다름" | "판단불가"
- `evidence`: 판단 근거를 구체적으로 나열 (배열)
- `suggested_stage_for_next_analysis`: 다음 분석에 적절한 단계 ("1"|"2"|"3"|"4"|"5"|"6"|"other")

[공통 분석 원칙]
- 상세 안전 분석 및 출력 형식은 공통 규칙을 따른다.

[출력 텍스트 작성 시 주의사항 (매우 중요)]
1. **내부 필드명 노출 금지**:
   - `evidence`, `summary`, `examples`, `comment` 등 유저가 읽는 텍스트 필드에 **JSON 필드명이나 기술 용어를 절대 포함하지 마시오.**
   - 금지 단어 예시: `advanced_behaviors`, `timeline_observations`, `behavior_summary`, `metadata`, `JSON`, `field`
   - ❌ 나쁜 예: "advanced_behaviors 필드에 언급된 대로 균형 감각이 뛰어납니다."
   - ✅ 좋은 예: "이 단계에서 기대되는 수준보다 더 뛰어난 균형 감각을 보여줍니다."

2. **자연스러운 문장**:
   - 분석 결과는 부모님이 읽는 보고서입니다. 딱딱한 데이터 나열이 아닌, 전문가가 설명해주는 듯한 자연스러운 문체로 작성하세요.


[development_insights 필드 정의]
⚠️ **중요**: `development_insights`는 발달 요약(`summary`)과 **명확히 구분**되는, 전문가적 해석과 판단이 담긴 인사이트입니다.

**summary vs insights 차이:**
- `summary` (요약): "무엇이 관찰되었는가" - 객관적 사실 서술
  - 예: "영상에서 아이는 주로 블록 쌓기를 했고, 9개월 월령에 적합한 손가락 협응력을 보입니다."
  
- `development_insights` (인사이트): "이것이 의미하는 바는 무엇인가" - 전문가적 해석과 조언
  - 예: "손가락 협응력이 또래 대비 빠르게 발달하고 있어, 더 복잡한 조작 놀이를 도입할 수 있습니다."

**작성 가이드라인:**
1. **비교와 평가**: 또래 대비, 이전 대비, 기대 수준 대비 평가
2. **발달 의미**: 관찰된 행동이 발달 과정에서 어떤 의미를 가지는지
3. **실행 가능한 조언**: 부모가 취할 수 있는 구체적 행동 제안

**핵심**: 가능한 경우 항상 2~3개의 구체적이고 실행 가능한 인사이트를 생성하세요. 정말로 인사이트를 도출할 수 없는 경우에만 빈 배열(`[]`)을 반환하십시오. `null` 값을 반환하지 마십시오.

⚠️ **길이 제한**: 각 인사이트는 **50자 이내**로 간결하게 작성하세요. 핵심 메시지를 명확히 전달하되 불필요한 수식어는 제거하세요.

**좋은 예시 (50자 이내):**
- "대근육 발달이 빠릅니다. 안전한 환경을 준비해주세요."
- "언어 표현이 적습니다. 말을 많이 걸어주세요."
- "집중력이 짧습니다. 한 가지 놀이를 함께 해보세요."

**나쁜 예시 (요약과 중복):**
- ❌ "블록 쌓기를 많이 했습니다." (단순 관찰 사실)
- ❌ "9개월 월령에 적합한 발달을 보입니다." (요약에 이미 있는 내용)
- ❌ "소근육 발달이 좋습니다." (구체성 부족)
