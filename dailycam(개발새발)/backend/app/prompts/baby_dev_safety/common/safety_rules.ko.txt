
[공통 분석 가이드: 안전 및 출력]

부정 분석 – 안전 분석(safety_analysis)

★ 이 단계에서는 반드시 다음 5가지를 생성해야 한다:
1) environment_risks  
2) critical_events  
3) incident_events (전체 발생 이벤트 기록)  
4) incident_summary (등급별 횟수 + 실제 감점)  
5) safety_score (최종 안전 점수)

------------------------------------
(1) environment_risks
------------------------------------
- 각 위험 요소에 대해 다음 필드를 상세히 기록한다:
  - risk_type: 낙상 / 충돌 / 끼임 / 질식·삼킴 / 화상 / 기타
  - severity: 사고 / 위험 / 주의 / 권장
  - trigger_behavior: 어떤 행동이 위험을 유발했는지
  - environment_factor: 환경적 요인(모서리보호대 없음, 미끄러운 바닥 등)
  - has_safety_device: true/false (안전장치 유무)
  - safety_device_type: 모서리보호대 / 서랍잠금장치 / 문스토퍼 / 계단안전문 / 미끄럼방지매트 / 기타
  - comment: 이유 및 개선점
- **작성 시 주의사항**: 앞서 제시된 `[이 단계에서 특히 중요한 안전사고 위험 포인트]` 섹션의 내용을 우선적으로 참고하여, 해당 월령에 특화된 위험을 식별한다.
- 보호자의 동반 여부와 개입 방식을 관찰하여 `adult_presence`와 `overall_safety_level`을 평가한다.

------------------------------------
(2) critical_events
------------------------------------
- 실제 사고 또는 사고 직전 상황(심각도 '사고' 또는 '위험')을 기록한다.
- event_type: 실제사고 / 사고직전위험상황
- timestamp_range: **반드시 HH:MM:SS-HH:MM:SS 형식으로 기록** (예: "00:00:05-00:00:10"). 비디오 전체 길이는 메타데이터의 video_total_time을 참고하라.
- description: 상황 설명
- estimated_outcome: 큰부상가능 / 경미한부상가능 / 놀람·정서적스트레스 / 기타

------------------------------------
(3) incident_events  (감점 계산용 - 전체 이벤트 기록)
------------------------------------
⚠️ **중요**: incident_events는 감점 로직을 위한 기초 데이터입니다.
- 영상에서 관찰된 모든 '사고·위험·주의·권장' 이벤트를 1개씩 전부 기록한다.
- **사용처**: safety_score 계산, incident_summary 생성
- **필수 필드**:
  - event_id: "E001", "E002" 등 순차 부여 (감점 계산 시 참조용)
  - severity: 사고 / 위험 / 주의 / 권장
  - timestamp_range: **반드시 HH:MM:SS-HH:MM:SS 형식으로 기록**
  - description: 상세 설명
  - has_safety_device: true/false (안전장치 유무 - 감점 계산에 영향)
- **주의**: incident_events와 safety_events는 별도로 생성해야 합니다. 
  incident_events는 감점용, safety_events는 타임라인 표시용입니다.

------------------------------------
(4) incident_summary  (등급별 횟수 + 감점 적용)
------------------------------------
각 severity 등급별로 다음을 기록한다:
- severity: 사고 / 위험 / 주의 / 권장
- occurrences: 해당 등급이 영상에서 총 몇 번 나타났는지 (incident_events에서 계산)
- applied_deduction: 실제 감점 값

------------------------------------
(5) 안전도 점수 계산 규칙 (safety_score)
------------------------------------

기본 점수 = 100점

감점 규칙:
- 사고: -50점 (최대 1회만 감점 적용)
- 위험: -30점 (최대 1회만 감점 적용)
- 주의: -10점 (최대 1회만 감점 적용)
- 권장: -2점 × 발생 건수 (최대 -16점)

계산 방식:
- incident_events → 모든 기록을 반영하여 각 등급별 occurrences 계산
- incident_summary → 감점 규칙에 따라 applied_deduction 생성
- safety_score = 100 + 모든 applied_deduction 합계
- 최종 점수는 50점보다 낮아지면 50점으로 처리한다.

------------------------------------
(6) safety_events (타임라인/DB 저장용 안전 이벤트)
------------------------------------
⚠️ **중요**: safety_events는 Dashboard 타임라인 표시 및 DB 저장을 위한 상세 데이터입니다.
- **사용처**: Dashboard 타임라인(빨간색/노란색), SafetyEvent 테이블 저장, SafetyReport 사고 유형 차트
- **incident_events와의 차이점**:
  - incident_events: 감점 계산용 (event_id, severity, has_safety_device 중심)
  - safety_events: UI 표시용 (title, location, description, resolved 중심)
- 각 이벤트는 다음 필드를 포함해야 한다:
  - severity: "위험" / "주의" / "권장" (타임라인 색상 결정)
  - title: 짧고 명확한 제목 (예: "주방 근처 접근", "낙상 위험"). **반드시 다음 카테고리 중 하나 이상을 포함하도록 노력하시오: "낙상", "충돌/부딛힘", "끼임", "전도(가구 넘어짐)", "감전", "질식", "화상"** - **필수**
  - description: 상세 설명 (예: "데드존에 3회 접근했습니다.") - **필수**
  - location: 발생 위치 (예: "주방 입구", "거실") - 장소별 통계용 - **필수**
  - timestamp_range: **반드시 HH:MM:SS-HH:MM:SS 형식** (예: "00:01:20-00:01:25") - **필수**
  - resolved: false (기본값, 해결 여부는 사용자가 나중에 체크)
- **주의**: safety_events는 incident_events와 별도로 생성하며, 
  같은 사건이라도 UI에 표시하기 좋은 형태로 title과 location을 명확히 작성해야 합니다.

------------------------------------
(7) safety_summary (안전 요약)
------------------------------------
⚠️ **중요**: `safety_summary`는 영상에서 관찰된 안전 상황을 **객관적으로** 서술하는 요약입니다.
- **목적**: 이 영상에서 어떤 안전 이벤트가 발생했고, 전체적인 안전 수준이 어떤지를 2-3문장으로 요약
- **작성 방법**:
  - 관찰된 주요 안전 이벤트 언급 (예: "영상에서 2건의 주의 상황이 발생했습니다")
  - 전반적인 안전 수준 평가 (예: "전반적으로 안전하게 활동했지만 주방 근처 접근에 주의가 필요합니다")
  - 객관적이고 중립적인 톤으로 작성
- **예시**:
  - "관찰 시간 동안 1건의 위험 상황(테이블 모서리 충돌 우려)과 2건의 주의 상황(주방 접근)이 발생했습니다. 전반적으로 보호자의 관리 하에 안전하게 활동했습니다."
  - "영상 전체에서 특별한 안전 이슈는 관찰되지 않았으며, 아이는 안전한 놀이 공간에서 활동했습니다."
- **주의**: `safety_insights`와 중복되지 않도록, 여기서는 '무엇이 발생했는가'에 초점을 맞추세요.
- Dashboard 메인 화면에 표시될 요약 텍스트

------------------------------------
(8) safety_insights (안전 인사이트)
------------------------------------
⚠️ **중요**: `safety_insights`는 안전 요약(`safety_summary`)과 **명확히 구분**되는, 전문가적 해석과 예방 조언이 담긴 인사이트입니다.

**safety_summary vs safety_insights 차이:**
- `safety_summary` (요약): "무엇이 발생했는가" - 객관적 사실 서술
  - 예: "2건의 주의 상황(주방 접근)이 발생했습니다. 전반적으로 안전하게 활동했습니다."
  
- `safety_insights` (인사이트): "이것이 의미하는 바와 대응 방법" - 전문가적 해석과 구체적 조언
  - 예: "주방 근처 접근이 반복되는 패턴이 보입니다. 이동식 안전 게이트를 설치하면 예방 가능합니다."

**작성 가이드라인:**
1. **패턴 분석**: 반복되는 위험 행동이나 안전 이슈 패턴 지적
2. **위험 평가**: 관찰된 상황이 얼마나 심각한지, 어떤 결과로 이어질 수 있는지
3. **구체적 해결책**: 부모가 즉시 실행할 수 있는 예방/개선 방법 제시

**핵심**: 가능한 경우 항상 2~3개의 구체적이고 실행 가능한 인사이트를 생성하세요. 정말로 인사이트를 도출할 수 없는 경우에만 빈 배열(`[]`)을 반환하십시오. `null` 값을 반환하지 마십시오.

⚠️ **길이 제한**: 각 인사이트는 **50자 이내**로 간결하게 작성하세요. 핵심 메시지를 명확히 전달하되 불필요한 수식어는 제거하세요.

**좋은 예시 (50자 이내):**
- "주방 접근이 반복됩니다. 이동식 게이트 설치를 권장합니다."
- "균형을 자주 잃습니다. 미끄럼 방지 매트를 깔아주세요."
- "가구 모서리가 위험합니다. 보호대 부착이 필요합니다."

**나쁜 예시 (너무 길거나 요약과 중복):**
- ❌ "2건의 주의 상황이 발생했습니다." (단순 사실 나열)
- ❌ "전반적으로 안전했습니다." (요약에 이미 있는 내용)
- ❌ "주의가 필요합니다." (구체성 부족, 실행 방법 없음)

------------------------------------
발달 분석 (development_analysis)
------------------------------------

★ 이 단계에서는 반드시 다음을 생성해야 한다:
1) summary (발달 요약)
2) main_activity (주요 활동)
3) development_score (발달 점수)
4) development_radar_scores (오각형 차트용 점수)
5) development_events (타임라인용 발달 이벤트)
6) skills (관찰된 발달 기술)
7) next_stage_signs (다음 단계 징후)

------------------------------------
(1) main_activity (주요 활동)
------------------------------------
- 영상에서 가장 많이 관찰된 활동을 한 줄로 기록
- 예: "블록 쌓기", "낮잠", "배밀이 연습"
- Dashboard 메인 화면에 표시

------------------------------------
(2) development_score (발달 점수)
------------------------------------
- 전체 발달 수준을 0-100점으로 평가
- 관찰된 발달 기술의 다양성, 빈도, 숙련도를 종합 평가
- DevelopmentReport에 표시

------------------------------------
(3) development_radar_scores (오각형 차트 점수)
------------------------------------
- 5개 영역별로 0-100점 평가:
  - "운동": 대근육/소근육 운동 발달
  - "언어": 언어 이해 및 표현
  - "인지": 인지 능력 및 문제 해결
  - "사회성": 타인과의 상호작용
  - "정서": 정서 표현 및 조절
- DevelopmentReport의 오각형 차트에 사용

------------------------------------
(4) development_events (타임라인용 발달 이벤트)
------------------------------------
- Dashboard 타임라인에 표시될 발달 이벤트를 기록한다.
- 각 이벤트는 다음 필드를 포함해야 한다:
  - category: "운동" / "언어" / "인지" / "사회성"
  - title: 짧고 명확한 제목 (예: "배밀이 연습 (15분)")
  - description: 상세 설명 (예: "대근육 발달 촉진")
  - is_sleep: false (수면 여부, true면 타임라인에서 '수면'으로 묶어서 표시)

------------------------------------
(5) summary (발달 요약)
------------------------------------
⚠️ **중요**: `summary`는 영상에서 관찰된 전반적인 발달 수준을 **객관적으로** 서술하는 요약입니다.
- **목적**: 이 영상에서 어떤 행동들이 관찰되었고, 전체적인 발달 수준이 어떤지를 2-3문장으로 요약
- **작성 방법**:
  - 관찰된 주요 발달 행동을 언급 (예: "영상에서 아이는 주로 배밀이와 잡고 일어서기 연습을 했습니다")
  - 전반적인 발달 수준 평가 (예: "7개월 월령에 적합한 대근육 운동 발달을 보이고 있습니다")
  - 객관적이고 중립적인 톤으로 작성
- **예시**:
  - "영상에서 아이는 주로 블록 쌓기와 소근육 활동에 집중했습니다. 9개월 월령에 적합한 손가락 협응력을 보이며, 양손을 사용한 물체 조작이 원활합니다."
  - "배밀이와 네발 기기를 활발하게 수행하며 7개월 월령의 전형적인 대근육 발달 수준을 보입니다. 가끔 잡고 일어서기를 시도하는 모습도 관찰되었습니다."
- **주의**: `development_insights`와 중복되지 않도록, 여기서는 '무엇이 관찰되었는가'에 초점을 맞추세요.
- DevelopmentReport에 표시될 메인 요약 텍스트

------------------------------------
추천 활동 (recommendations)
------------------------------------
- 아이의 현재 발달 수준에 맞는 놀이 활동을 추천한다.
- 각 추천은 다음 필드를 포함해야 한다:
  - title: 활동 이름 (예: "까꿍 놀이", "블록 쌓기")
  - benefit: 발달 영역 (예: "인지", "소근육", "언어", "사회성")
- 2-5개 정도 추천

------------------------------------
하이라이트 클립 (highlight_clips)
------------------------------------
- 영상에서 특별히 주목할 만한 순간을 클립으로 추천한다.
- **목적**: 부모가 중요한 순간을 빠르게 확인할 수 있도록 3-5개의 하이라이트 순간을 추출
- **선정 기준**:
  - 발달 이정표 달성 (처음으로 성공한 행동, 중요한 발달 기술 등)
  - 안전 관련 주요 이벤트 (위험 상황, 주의가 필요한 순간)
  - 특별히 인상적이거나 의미 있는 순간

각 클립은 다음 필드를 포함해야 한다:
  - **title**: 클립 제목 (짧고 명확하게, 예: "배밀이 2미터 이동 성공!", "주방 근처 접근 3회") - **필수**
  - **description**: 해당 순간에 대한 상세 설명 (1-2문장, 왜 이 순간이 중요한지 설명) - **필수**
  - **category**: "발달" / "안전" (탭 구분용) - **필수**
  - **sub_category**: 세부 분류 (예: "운동 발달", "언어 발달", "주방 접근", "낙상 위험") - **필수**
  - **importance**: "high" / "medium" / "low" (중요도) - **필수**
  - **timestamp_range**: 해당 순간의 시작-종료 시간 (HH:MM:SS-HH:MM:SS 형식, 예: "00:05:20-00:05:35") - **필수**
  
  ⚠️ **주의**: 
  - `video_url`과 `thumbnail_url`은 시스템에서 자동으로 생성하므로 **포함하지 마세요**.
  - `duration_seconds`는 `timestamp_range`에서 자동으로 계산됩니다.

- **권장 개수**: 3-5개 (너무 많으면 5개로 제한)
- **우선순위**: 
  1. 발달 이정표 > 2. 안전 위험 이벤트 > 3. 특별한 순간

디스클레이머
- `disclaimer` 필드에 아래 문장을 반드시 포함한다:
  "이 평가는 영상 기반 관찰에 따른 참고용 발달·안전 분석이며, 의학적 진단이나 전문 평가를 대체하지 않습니다."

- JSON 외 텍스트(마크다운 코드블록 ```json 등)는 절대 포함하지 않는다.

⚠️ **JSON 출력 시 필수 준수 사항 (파싱 오류 방지):**
1. **Trailing Comma 금지**: 배열이나 객체의 마지막 항목 뒤에 쉼표(,)를 절대 붙이지 마세요.
   - ❌ 잘못된 예: `{"key": "value",}` 또는 `["item1", "item2",]`
   - ✅ 올바른 예: `{"key": "value"}` 또는 `["item1", "item2"]`
2. **문자열 내 따옴표 이스케이프**: 문자열 안에 큰따옴표(")가 있으면 반드시 `\"`로 이스케이프하세요.
3. **줄바꿈 이스케이프**: 문자열 안의 줄바꿈은 `\n`으로 표현하세요.
4. **유효한 JSON만 출력**: 주석, 설명, 기타 텍스트를 JSON 밖에 절대 포함하지 마세요.
