# HLS 지연 시간(Latency) 및 스트리밍 특성

## 1. HLS (HTTP Live Streaming) 개요
DailyCam은 실시간 모니터링을 위해 Apple이 개발한 **HLS 프로토콜**을 사용합니다.
*   **방식**: 영상을 잘게 쪼갠 조각 파일(`.ts`)을 HTTP로 다운로드하여 재생.
*   **장점**: 방화벽 친화적, 모바일/PC 호환성 최강, 끊김 없는 품질 자동 조절.
*   **단점**: 태생적으로 **지연 시간(Latency)**이 발생함. (WebRTC나 RTSP보다 느림)

---

## 2. 지연 시간 발생 원리 (Fact)

### 지연 시간 계산 공식
$$ \text{Total Latency} \approx (\text{Segment Duration} \times \text{Buffer Count}) + \text{Encoding/Network Delay} $$

### 현재 코드 설정 (`backend/app/services/live_monitoring/hls_stream_generator.py`)
현재 DailyCam은 지연 시간을 줄이기 위해 **"Low Latency 튜닝"**이 적용되어 있습니다.

```python
# FFmpeg 옵션 설정값
'-hls_time', '2',       # 각 조각(.ts) 길이: 2초 (매우 짧음)
'-hls_list_size', '5',  # 재생 목록(.m3u8)에 유지할 개수: 5개
'-hls_flags', 'delete_segments+append_list',
```

### 예상 지연 시간
*   **세그먼트 길이**: 2초
*   **플레이어 버퍼링**: 보통 안정적 재생을 위해 3개 정도를 미리 받음. (2초 × 3 = 6초)
*   **결과**: 사용자에게는 **약 6~10초** 정도의 과거 영상이 보입니다.

## 3. 트러블슈팅 및 방어 논리

### Q. "실시간이라더니 왜 10초 느려요?"

**A. 기술적 선택에 따른 의도된 설계입니다.**
1.  **안정성 우선**: 1초 미만의 초저지연(WebRTC)은 네트워크가 조금만 불안해도 영상이 끊기거나 뭉개집니다. 육아 모니터링은 **"끊기지 않고 안정적으로 보는 것"**이 더 중요합니다.
2.  **보편적 호환성**: 별도의 플러그인 없이 아이폰, 안드로이드, PC 모든 브라우저에서 재생하려면 HLS가 표준입니다.
3.  **최적화 적용됨**: 일반적인 HLS(10초 청크 사용 시 30초 지연)보다 훨씬 빠른 **2초 튜닝**을 적용하여 업계 표준 대비 상당히 빠른 편입니다.

### 더 줄일 수 없나요?
*   `hls_time 1` (1초)로 줄일 수 있으나, 서버 CPU 부하가 심해지고 네트워크 오버헤드가 커져서 끊김이 발생할 확률이 큽니다. **현재의 2초 설정이 안정성과 속도의 최적 타협점(Sweet Spot)**입니다.
