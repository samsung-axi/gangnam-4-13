# 이슈 해결 보고서: 비디오 플레이어 중복 렌더링으로 인한 리소스 누수

**날짜:** 2025-12-15
**작성자:** Antigravity (AI Assistant)
**상태:** ✅ 해결됨

---

## 1. 🚨 문제 상황 (Symptoms)

사용자가 모니터링 페이지(`Monitoring.tsx`)에 진입했을 때, 화면 상단에 **의도치 않은 검은색 비디오 플레이어**가 표시되고, 그 아래에 정상적인 모니터링 화면(플레이어 포함)이 표시되는 현상이 발생함.

- **증상**: 화면에 동일한 라이브 스트림을 재생하는 플레이어가 **2개** 동시에 존재.
- **시각적 오류**: 레이아웃 상단에 불필요한 비디오 영역이 겹치거나 밀려 있음.

## 2. 🔍 원인 분석 (Root Cause)

**`AppLayout.tsx`와 `Monitoring.tsx`의 역할 중복**이 원인이었습니다.

1.  **`AppLayout.tsx` (레이아웃 컴포넌트)**
    - "페이지를 이동해도 비디오가 끊기지 않게 유지(Keep-Alive)"하려는 의도로 `HLSVideoPlayer`를 전역적으로 렌더링하고 있었습니다.
    - 특히 `isMonitoringPage`일 때 상단에 플레이어를 표시하도록 코딩되어 있었습니다.

    ```tsx
    // AppLayout.tsx (수정 전)
    {isMonitoringPage && hlsUrl && (
        <div className="w-full bg-black ...">
            <HLSVideoPlayer ... />
        </div>
    )}
    ```

2.  **`Monitoring.tsx` (페이지 컴포넌트)**
    - 모니터링 페이지 자체적으로 이미 완벽한 `HLSVideoPlayer` UI와 로직을 구현하고 있었습니다.

**결과적으로 사용자가 모니터링 페이지에 들어가면:**
- `AppLayout`이 만든 플레이어 1개 (상단)
- `Monitoring` 페이지가 만든 플레이어 1개 (하단)
👉 **총 2개의 플레이어가 동시에 돌아가는 상황 발생.**

## 3. ⚠️ 리소스에 미친 영향 (Impact)

이 "이중 렌더링"은 단순한 UI 버그를 넘어 심각한 **리소스 낭비**를 초래했습니다.

| 리소스 구분 | 영향 | 세부 내용 |
| :--- | :--- | :--- |
| **네트워크 (대역폭)** | **200% 사용 (2배)** | 동일한 HLS 세그먼트(`.ts`) 파일들을 두 플레이어가 각각 서버로부터 계속 다운로드함. |
| **서버 부하** | **증가** | FastAPI 백엔드로 들어오는 요청 수가 2배로 증가 (불필요한 트래픽). |
| **클라이언트 CPU** | **낭비** | 브라우저가 두 개의 고행상도 비디오를 동시에 디코딩하느라 CPU/GPU 점유율 상승. |
| **메모리/배터리** | **낭비** | 불필요한 DOM 노드 및 JS 인스턴스 유지. 모바일 기기의 경우 배터리 소모 가속화. |

## 4. 🛠️ 해결 방법 (Resolution)

**`AppLayout.tsx`에서 불필요한 전역 플레이어 렌더링 로직을 완전히 삭제했습니다.**

- `Monitoring.tsx` 페이지가 이미 플레이어를 책임지고 있으므로, 레이아웃 차원에서 중복으로 띄워줄 필요가 없다고 판단했습니다.
- "페이지 간 이동 시 재생 유지(Keep-Alive)" 기능보다 **리소스 효율성과 UI 안정성**을 최우선으로 하여 과감히 제거했습니다.

### 수정 코드 (`frontend/src/components/layout/AppLayout.tsx`)

**변경 전:**
```tsx
// 전역 플레이어 렌더링 코드 존재
{isMonitoringPage && hlsUrl && ( ... )}
{!isMonitoringPage && hlsUrl && ( ... )}
```

**변경 후:**
```tsx
// 👉 모든 HLSVideoPlayer 렌더링 코드 삭제
// 깔끔하게 Outlet만 렌더링
<Outlet context={{ hlsUrl, setHlsUrl, ... }} />
```

## 5. ✅ 결과 (Result)

- 모니터링 페이지 상단의 **이상한 검은색 비디오 영역 사라짐**.
- **플레이어가 딱 1개만 실행됨**.
- 네트워크 트래픽 및 클라이언트 리소스 사용량 **정상화 (50% 감소)**.
- UI가 의도한 대로 깔끔하게 표시됨.
