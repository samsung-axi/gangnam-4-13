# Gemini 비디오 분석 비용 효율성 진단 보고서

## 1. 개요
현재 DailyCam의 비디오 분석 파이프라인(Gemini 2.5 Flash 기반)의 비용 구조를 진단하고, 24시간 모니터링 시나리오에서의 예상 비용과 현재 적용된 코드 레벨의 최적화 현황을 정리한 문서입니다.

---

## 2. 비용 구조 분석 (Cost Structure)

Gemini API의 비디오 분석 요금은 **입력 비디오의 길이(분)**에 비례합니다. (해상도보다 길이에 민감)

*   **모델**: Gemini 2.5 Flash
*   **단가**: 약 **$0.075 / 분** (입력 토큰 기준 추산)

### 24시간 모니터링 시뮬레이션
만약 필터링 없이 24시간(1440분) 영상을 모두 분석할 경우:

*   **1일 비용**: 1440분 × $0.075 $\approx$ **$108.00**
*   **1개월(30일) 비용**: $108 × 30 $\approx$ **$3,240.00**

> ⚠️ **진단 결과**: 현재의 "전수 조사(Full Analysis)" 방식은 상시 녹화 서비스용으로 비용 효율성이 매우 낮습니다.

---

## 3. 현재 코드 적용 최적화 (Implemented Optimization)

현재 코드(`backend/app/services/gemini_service.py`)에는 다음과 같은 최적화가 적용되어 있습니다.

### 3-1. 전처리 다운샘플링 (480p / 15fps)
```python
# _optimize_video 메서드
'-vf', f'scale={target_width}:{target_height},fps={target_fps}' # 480p, 15fps
```
*   **효과**: 
    *   업로드 파일 용량 대폭 감소 (네트워크 대역폭 절약)
    *   API 처리 대기 시간 단축
*   **한계**: Gemini는 '비디오 길이(시간)'로 과금하므로, FPS나 해상도를 줄여도 **직접적인 API 과금 절감 효과는 미미**합니다. (시스템 리소스 절약용)

### 3-2. 스토리지 비용 제로화 (Storage Zeroization)
```python
# analyze_video_vlm 메서드
await asyncio.to_thread(genai.delete_file, video_file.name)
```
*   **효과**: 분석 즉시 Google Cloud Storage 파일을 삭제하여, 추가적인 스토리지 과금을 **완벽하게 차단**했습니다.

## 4. 제언 및 향후 과제 (Recommendations)

현재의 10분 단위 세그먼트 분석 구조는 유지하되, **비용 폭탄**을 막기 위해 다음 단계의 고도화가 필수적입니다.

### 4-1. 움직임 기반 필터링 (Motion Filtering) 도입 필요
현재 프로젝트에는 `opencv-python`이 이미 포함되어 있습니다. 이를 활용해 **"움직임이 없는 구간(Dead Time)"**을 감지하여 API 호출을 Skip 하는 로직 추가가 시급합니다.
*   예상 절감 효과: 가정집 기준 약 **60~70%** (수면, 외출 시간 등)

### 4-2. 프레임 추출 방식 검토 (Frame Sampling)
동영상 업로드 방식 대신, 1분당 1~2장의 핵심 프레임(이미지)만 추출하여 전송하는 방식으로 전환을 고려해야 합니다.
*   비디오($0.075/분) vs 이미지($0.00025/장)
*   **비용 절감 효과**: 이론상 **90% 이상** 절감 가능.
