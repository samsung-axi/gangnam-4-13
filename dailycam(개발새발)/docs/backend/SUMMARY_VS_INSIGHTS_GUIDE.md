# Summary vs Insights 구분 가이드

## 📋 개요

VLM 분석 결과에서 `summary` (요약)와 `insights` (인사이트)가 명확히 구분되지 않아 중복된 내용이 발생하는 문제를 해결하기 위한 가이드입니다.

## 🎯 핵심 차이점

### Summary (요약) - "무엇이 관찰되었는가"
- **목적**: 객관적 사실 서술
- **톤**: 중립적, 관찰 중심
- **내용**: 영상에서 실제로 일어난 일
- **길이**: 2-3문장

### Insights (인사이트) - "이것이 의미하는 바는 무엇인가"
- **목적**: 전문가적 해석과 조언
- **톤**: 분석적, 조언 중심
- **내용**: 관찰의 의미, 비교, 실행 가능한 제안
- **개수**: 2-3개 (각 1-2문장)

---

## 📊 발달 분석 (Development Analysis)

### ✅ 좋은 예시

#### `development_analysis.summary` (발달 요약)
```
영상에서 아이는 주로 블록 쌓기와 소근육 활동에 집중했습니다. 
9개월 월령에 적합한 손가락 협응력을 보이며, 양손을 사용한 물체 조작이 원활합니다.
```
→ 객관적으로 "무엇을 했고, 어떤 수준인가"를 서술

#### `development_analysis.development_insights` (발달 인사이트)
```json
[
  "손가락 협응력이 또래 대비 빠르게 발달하고 있어, 구슬 끼우기나 퍼즐 같은 더 복잡한 조작 놀이를 도입할 수 있습니다.",
  "집중력이 한 가지 활동에 5분 이상 유지되어 매우 긍정적입니다. 이는 인지 발달의 좋은 신호입니다."
]
```
→ "이것이 의미하는 바"와 "부모가 무엇을 할 수 있는가"를 제시

### ❌ 나쁜 예시 (중복)

#### Summary에 인사이트가 섞여 있음
```
영상에서 아이는 블록 쌓기를 했습니다. 
손가락 협응력이 뛰어나므로 더 복잡한 놀이를 도입할 수 있습니다.
```
→ ❌ 두 번째 문장은 insights에 들어가야 함

#### Insights가 단순 사실 나열
```json
[
  "블록 쌓기를 많이 했습니다.",
  "9개월 월령에 적합한 발달을 보입니다."
]
```
→ ❌ Summary와 중복, 해석이나 조언이 없음

## 🛡️ 안전 분석 (Safety Analysis)

### ✅ 좋은 예시

#### `safety_analysis.safety_summary` (안전 요약)
```
관찰 시간 동안 1건의 위험 상황(테이블 모서리 충돌 우려)과 2건의 주의 상황(주방 접근)이 발생했습니다. 
전반적으로 보호자의 관리 하에 안전하게 활동했습니다.
```
→ 객관적으로 "무엇이 발생했는가"를 서술

#### `safety_analysis.safety_insights` (안전 인사이트)
```json
[
  "주방 근처 접근이 3회 반복되는 패턴이 보입니다. 아이의 이동 반경이 넓어지는 시기이므로 이동식 안전 게이트 설치를 권장합니다.",
  "테이블 모서리 충돌 위험이 있었으나 다행히 부상은 없었습니다. 가구 모서리 보호대를 부착하면 안전사고를 예방할 수 있습니다."
]
```
→ "패턴 분석"과 "구체적 해결책"을 제시

### ❌ 나쁜 예시 (중복)

#### Summary에 조언이 섞여 있음
```
2건의 주의 상황이 발생했습니다. 
안전 게이트를 설치하면 좋겠습니다.
```
→ ❌ 두 번째 문장은 insights에 들어가야 함

#### Insights가 단순 사실 나열
```json
[
  "2건의 주의 상황이 발생했습니다.",
  "전반적으로 안전했습니다."
]
```
→ ❌ Summary와 중복, 해석이나 해결책이 없음

## 📐 작성 규칙

### Summary 작성 시
1. ✅ 관찰된 사실만 서술
2. ✅ "영상에서", "관찰되었습니다" 등 객관적 표현 사용
3. ✅ 수준 평가는 OK (예: "9개월 월령에 적합함")
4. ❌ 조언이나 제안 금지
5. ❌ "~하면 좋습니다", "~를 권장합니다" 금지

### Insights 작성 시
1. ✅ 전문가적 해석 포함
2. ✅ 비교 표현 (예: "또래 대비", "이전 대비")
3. ✅ 실행 가능한 조언 제시
4. ✅ "~하면 도움이 됩니다", "~를 권장합니다" 사용
5. ❌ 단순 사실 나열 금지
6. ❌ Summary와 중복 금지

## ✅ 기대 효과

1. **중복 제거**: Summary와 Insights가 서로 다른 정보를 제공
2. **가독성 향상**: 부모가 빠르게 요약을 읽고, 필요시 인사이트를 확인
3. **실용성 증가**: 인사이트에서 실행 가능한 조언 제공
4. **전문성 강화**: 전문가적 해석과 분석이 명확히 드러남

## 📱 UI에서의 활용

### Development Report
- **큰 카드 (Summary)**: 
  - "오늘의 발달 요약" 섹션에 표시
  - 빠르게 훑어볼 수 있는 객관적 요약
  
- **작은 박스 (Insights)**:
  - "발달 인사이트" 섹션에 bullet point로 표시
  - 전문가 조언이 필요한 경우 확인

### Safety Report
- **Summary**: 안전 상황 한눈에 파악
- **Insights**: 구체적 예방 조치 확인

## 🔍 검증 방법

### 프롬프트 테스트 시 확인사항

1. **Summary 검증**:
   - [ ] 객관적 사실만 포함되어 있는가?
   - [ ] 조언이나 제안이 없는가?
   - [ ] 2-3문장으로 적절한가?

2. **Insights 검증**:
   - [ ] Summary와 내용이 다른가?
   - [ ] 전문가적 해석이 포함되어 있는가?
   - [ ] 실행 가능한 조언이 있는가?
   - [ ] 2-3개로 적절한가?

3. **전체 검증**:
   - [ ] Summary만 읽어도 전체 상황 파악이 되는가?
   - [ ] Insights만 읽어도 무엇을 해야 할지 명확한가?
   - [ ] 두 개를 함께 읽으면 완전한 이해가 되는가?
