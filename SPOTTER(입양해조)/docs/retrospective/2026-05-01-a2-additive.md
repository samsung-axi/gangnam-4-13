## 4. 진단 — 왜 additive 가 0인가

### A. LGBM 의 implicit ensemble
- `industry_prior_pred` 가 LGBM 의 16번째 feature
- LGBM (`num_leaves=31, n_estimators=200`) 이 deep tree split 으로 industry-level signal 을 dong-level closure_rate_lag1 과 함께 학습
- additive 추가 = same signal 의 redundant linear combination
- LGBM 이 이미 더 표현력 높게 학습 중 → linear additive 가 개선 못 함

### B. Grid 의 한계
- Step 0.05 → 11 candidates [0.0, 0.05, ..., 0.50]
- 모두 baseline 미만 → not just no improvement, ALL DEGRADED
- 어떤 w_industry > 0 도 baseline 보다 나쁨 → strong evidence of redundancy

### C. Scale 보정 (X 2) 의 의미
- industry_prior_pred 는 0~0.5 (mean closure_rate prediction)
- ensemble_val_proba 는 0.07~0.56 (binary classifier proba)
- X 2 scale 보정 후에도 distribution 다름
- 만약 sigmoid transform 으로 더 정교하게 변환했다면 다른 결과 가능 — 별도 spec 가능

### D. Additive 의 이론적 정당성 vs 실제
- Wolpert (1992) stacking — 여러 모델의 별도 학습 후 meta-learner 가 가중
- 본 spec 의 additive 는 single grid-search 가중 → simple average → over-simplification
- 진짜 stacking 은 별도 meta-learner LGBM 필요 (현재는 단순 weighted sum)

## 6. 배운 점

1. **Grid search 결과의 정직한 해석** — "w=0" 은 단순 효과 없음이 아니라 **"current model 이 이미 최적 해 근처"** 의미. LGBM 의 implicit learning capacity 검증
2. **Linear additive 의 한계** — explicit additive 가 implicit non-linear (LGBM tree split) 보다 약함. 같은 신호의 linear 재조합은 noise 만 추가
3. **Sprint 의 fail-fast 가치** — grid search 자동 skip → production 무영향 → 빠른 결론. cfg flag + grid search 자동화의 패턴 가치
4. **하루 7 sprint 의 한계 신호** — 5 KEEP (E/C/D/A/B-3) + 3 rollback (B-1, D-3, A-2). 다음 시도들이 모두 marginal 또는 negative → **데이터 영역이 본질적 bottleneck**
5. **Wolpert stacking 이론 vs 단순 implementation** — 진짜 two-stage stacking 은 meta-learner 학습 필요. 단순 weighted sum 은 stacking 의 약식 버전 — 효과 약함
