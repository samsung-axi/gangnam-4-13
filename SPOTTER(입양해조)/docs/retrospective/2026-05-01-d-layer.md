# 2026-05-01 D layer 회고 — closure_risk threshold + top-K API

> **참조**: 본 회고는 같은 날 두 번째 sprint. C layer 회고는 별도 파일 `docs/retrospective/2026-05-01.md`.

## 요약

폐업 위험도 모델의 의사결정 layer 를 두 축으로 개선:
1. **Hard threshold (0.65/0.40) → val proba quantile 자동 fit** — 학습 시 ensemble val proba 의 q90 (danger) / q70 (caution) 을 metrics.json 에 저장. predict.py 의 `_load_risk_levels` 가 lru_cache 로 동적 load.
2. **`predict_topk(targets, k_pct)` API 추가** — 다수 (dong × industry) 조합에서 위험도 top K% 추천. P@K metric 과 직접 매핑되는 ranking decision.

5 task subagent + inline (T4 retrain + regression fix) 로 진행. **결과: fit threshold danger=0.4100 / caution=0.3026** 적용 (기존 hard 0.65/0.40 → 분포에 맞게 자동 조정). production proba 분포가 [0.15, 0.55] 범위에 갇혀 있던 문제 해결 — 이제 "danger" 라벨이 실제로 발생.

## 1. 배경

C layer fix (오전, commit 7e2e2ab) 후 ensemble test AUC 0.5983 으로 의미 있는 신호 회복. 하지만 추론 의사결정 layer 에는 두 문제 잔존:

### 문제 1: hard threshold 가 분포와 mismatch
`predict.py:26-30`:
```python
RISK_LEVELS = [(0.65, "danger"), (0.40, "caution"), (0.00, "safe")]
```

production proba 분포 (`metrics.json` test set):
- bin n_per_bin = `[0, 43, 197, 214, 146, 20, 0, 0, 0, 0]`
- 0.65 이상 bin: **0건** → "danger" 라벨 production 에서 0건
- 학술 근거 없는 임의 설정

### 문제 2: 사용자 의사결정 use case 미지원
- 모델 AUC ~0.60 수준이라 절대값 보다 **순위 (ranking)** 가 더 신뢰. P@10 = 0.30 (random 0.22 대비 1.36× 리프트)
- 사용자 (창업주): "내 동네에서 가장 위험한 가게 어디?" → top-K 추천 직관적
- 기존 단일 `predict()` API 는 1건씩만 — 비효율 + use case 미스매치

**가설**: threshold 자동 fit + top-K API 동시 도입 시 (1) production 에서 "danger" 라벨이 의미 있게 발화, (2) 호출자 (frontend, simulation) 의 ranking use case 직접 지원.

## 5. 배운 점

1. **Threshold 자동 fit 의 즉각적 가치** — hard 0.65/0.40 → fit 0.41/0.30 이라는 단일 변경으로 production 에서 "danger" 라벨이 0건 → 10% 로 회복. 모델 학습 비용 X, 재배포만 필요.
2. **Top-K reframe 의 학술적 정당성** — AUC 0.60 모델에서 binary classification 으로 차단 결정은 위험. ranking task (P@K, R@K) 로 reframe 하면 동일 신호에서 더 신뢰할 수 있는 의사결정 가능. P@10 1.36× lift 가 production 가치.
3. **`lru_cache(maxsize=1)` 의 함정** — process 내 학습 → 추론 분리 환경에서는 OK, 같은 process 에서 학습 후 추론 시 stale. test 에서 `cache_clear()` 명시적 호출 필요.
4. **Regression test 의 monkeypatch 격리 패턴** — fit threshold 가 default 와 달라 회귀가 깨짐. monkeypatch (WEIGHTS_DIR=tmp) 로 default fallback 강제 → 두 mode 모두 별도 test 로 검증 가능. **dynamic threshold 도입 시의 일반 패턴**.
5. **부수효과 적은 fix 의 안전성** — D layer 는 모델 학습 자체에 영향 X (학습 후처리만 변경). retrain 후 metric 변화 ±0.003 (랜덤 seed 노이즈) 로 본질적 동일. **stable refactor**.
6. **C 와 D 를 같은 날 연속 진행한 효과** — C 가 신호 깨끗하게 만든 직후 D 로 의사결정 layer 다듬음. C 가 만든 분포 (0.15~0.55) 에 D 의 quantile 자동 fit 이 자연스럽게 적용됨. **layer 순서: 신호 → 의사결정 흐름이 sprint 단위로도 유효**.
