## 4. 진단 — 왜 실패했나

### A. shift(4) NaN → 0 fallback 의 신호 왜곡
- `weekday_sales_yoy = (current - lag4) / (|lag4| + 1)` 가 첫 4분기 row 에서 NaN
- LGBM 입력 추출 시 `fillna(0)` → "yoy=0" 과 "yoy 모름" 의 혼동
- LGBM 이 split gain 으로 0 vs 비제로 를 split → train data 의 첫 4분기 패턴 학습 → val/test 일반화 X

### B. age ratio 의 약한 예측력
- `age_20_sales_ratio = age_20_sales / monthly_sales` — 단순 분배 비율
- 폐업과 직접 인과 약함. 트렌디 업종 비중이 높다고 폐업 위험 ↑ 라는 가설 노이즈 가능성
- 마포 16동 데이터에서 분배 비율 분산이 작아 split gain 미미

### C. macro 신호 (cpi_yoy, holiday_count) 의 mismatch
- cpi_index 는 전국 단일 값 → dong×industry row 마다 동일
- holiday_count 는 분기별 단일 값 → 같은 분기의 모든 dong row 동일
- 행정동 단위 fine-grained 데이터에 macro 신호 broadcast = 정보량 zero, noise 만 추가

### D. LGBM 의 약한 신호 split overfitting
- 신규 feature 의 split gain 이 약해도 LGBM (`num_leaves=31, n_estimators=200`) 은 train 의 미세 패턴 잡으려 split 수행
- 결과: train acc 약간 ↑, val/test acc 하락 (overfit)
- C/D layer 에서 깨끗했던 신호가 noise 추가로 generalization 망가짐

### 종합
신규 feature 8개 중 **유효한 신호가 거의 없음**. 도메인 정당성 (yoy = 변화율, ratio = 비중) 은 말이 되나 **현재 데이터/모델 조합에서는 실현되지 않음**.

## 6. 배운 점

1. **"잠자는 신호" 가설의 한계** — RDS 에 컬럼이 있다고 의미 있는 신호인 건 아님. 도메인 정당성만으로는 부족, 실제 split gain / lift 측정 필수.
2. **Macro 신호 broadcast 의 무용성** — 전국 cpi, 분기 holiday 를 행정동 row 마다 broadcast 하면 정보 0. local 신호여야 의미 있음.
3. **shift(4) NaN→0 의 함정** — yoy 계산의 첫 4분기는 fundamentally 알 수 없음. 0 fallback 이 "실제 0" 과 혼동되어 신호 왜곡. 더 정직한 처리: NaN row drop 또는 별도 indicator column.
4. **LGBM 의 split gain 과 overfit** — 약한 신호도 split 하려는 LGBM 의 특성. train AUC 약간 ↑ 면서 val/test 하락. early_stopping + leaf_count 제한 외 feature selection (e.g., SHAP 기반 후보 압축) 검토 필요.
5. **Production rollback 패턴 정립** — sprint 결과가 나쁠 때 weight 만 복원하고 코드는 보존 (안 B with safety). 다음 sprint 의 ablation 자료 보호. git checkout `<sha>` -- 로 selective rollback.
6. **"신호 → 의사결정 → 더 많은 신호" 의 순서** — C (label) ✅ → D (threshold) ✅ → B (feature) ❌. C/D 의 성공이 B 의 성공을 보장하진 않음. 각 layer 가 독립 검증 필요.
7. **회귀 test 가 rollback 안전 보장** — 40/40 test 가 rollback 후에도 통과 → predict.py 호환성 + LGBM_FEATURES 길이 검증. test 가 production 안전망 역할.
