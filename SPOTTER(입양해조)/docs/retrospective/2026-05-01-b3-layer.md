### 코드 변경

```python
# models/closure_risk/data_prep.py
def add_dong_residual_feature(df, train_quarters):
    train_df = df[df["quarter"].isin(train_quarters)]
    industry_mean = train_df.groupby("industry_code")["closure_rate_lag1"].mean()
    global_mean = float(train_df["closure_rate_lag1"].mean())
    df = df.copy()
    df["_industry_mean_lag1"] = df["industry_code"].map(industry_mean).fillna(global_mean)
    df["dong_closure_rate_residual_lag1"] = df["closure_rate_lag1"].fillna(0) - df["_industry_mean_lag1"]
    df = df.drop(columns=["_industry_mean_lag1"])
    return df

# train.py — Stage 1 호출 직후 통합
if cfg.get("enable_b3_dong_residual", True):
    df_labeled = add_dong_residual_feature(df_labeled, train_quarters)
```

### 평가
- **Test AUC 0.6149** — marginal +0.001. statistical noise 대비 의미 있는지 의문. 단 baseline 미만으로 떨어지진 X (의사결정 규칙 충족)
- **Test PR-AUC +0.0157** — 가장 큰 개선. imbalanced data 에서 더 의미 있는 metric. baseline pos_ratio 0.32 대비 lift 1.03x

- **Test P@10 -0.022** — trade-off. top 10% 의 precision 약간 하락. 단 R@10 동일
- **Val 영향 거의 X** — feature 가 train fit 만 영향, val/test 의 generalization 측정값 안정

## 4. 진단 — 왜 marginal 인가

### 가능한 원인
1. **LGBM 이 이미 implicit residual 학습** — `closure_rate_lag1` + Stage 1 `industry_prior_pred` 두 feature 의 조합으로 LGBM 이 deep tree split 으로 residual 추출 가능. 명시적 residual 추가는 redundant.
2. **predict.py 의 0 fallback** — production inference 에서 `dong_closure_rate_residual_lag1` 이 0 으로 들어감 (train fit 한 industry mean 미저장). 학습 시점만 효과 — production 효과 X
3. **AUC 0.60 plateau 고착** — 모델 / 데이터 의 fundamental 한계. feature engineering 으로 큰 개선 어려움
4. **PR-AUC 의 더 큰 변화** — imbalanced data 에서 ranking-based metric 이 작은 신호도 잘 반영. 이 부분만 의미 있는 개선

### B-1 과의 패턴 차이
- B-1 (8 feature): test AUC -0.024 (큰 degradation)
- B-3 (1 feature, hierarchical): test AUC +0.001 (marginal)
- 차이: B-1 의 single-set fit shift(4) NaN 패턴 vs B-3 의 train-only industry mean fit
- **단일 hierarchical feature 가 8 단순 derivation 보다 안전** — feature 개수 줄이고 명확한 의미 도입의 가치

## 6. 배운 점

1. **단일 hierarchical feature 의 안전성** — 8 단순 derivation (B-1) 보다 1 명확한 residual (B-3) 이 더 안정적. **feature engineering 은 양보다 의미가 우선**
2. **Marginal 결과의 가치** — +0.001 AUC 는 statistical noise 에 가까울 수 있지만, **+0.016 PR-AUC** 는 의미 있는 신호. 단일 metric 으로 판단 금지
3. **Train fit ↔ production inference 의 gap** — train.py 가 train_quarters 의 industry_mean 으로 fit 한 residual 을 production 에 적용하려면 lookup pkl 필요. 단순 0 fallback 은 학습 시 신호 학습만 (production 무영향). **B-3 expansion 의 자연스러운 다음 단계**
4. **AUC plateau 의 의미** — 0.6149 (오늘 6 sprint 끝)는 거의 동일 한 0.6139 (3 sprint 전 A). **모델 한계가 명확** — B-4 (새 데이터) 만이 본질적 돌파 후보
5. **하루 6 sprint 의 적절성** — B-3 까지 안전하게 KEEP 도달. 더 이상 시도 시 fatigue + rollback risk 증가. **여기서 break 가 적절**
