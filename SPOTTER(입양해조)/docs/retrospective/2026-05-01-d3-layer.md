## 4. 진단 — 왜 실패했나

### A. Val 단일 fit 으로 overfit
- IsotonicRegression 은 비-parametric — train 데이터 (val) 의 모든 점을 통과시킴
- 626 sample 의 val 분포에 fit → test 분포 (다른 분기, 다른 dong×industry mix) 와 mismatch
- **AUC 가 monotonic 하에 보존되어야 하지만, test 의 raw proba 가 train 외 범위로 나가면 clip → ranking 변경**
- → ranking 미세 변경 → AUC -0.04

### B. Step function 의 plateau 생성
- isotonic 의 fit 결과가 piecewise constant — 같은 raw proba bucket 의 모든 sample 이 같은 calibrated 값
- val 의 majority bucket 이 caution (0.30) 과 danger (0.40) 사이 plateau 에 잡힘
- q70 과 q90 모두 그 plateau 에 떨어져 같은 값

### C. Calibration 의 위장된 효과
- Val Brier 개선 (0.17 → 0.16) 은 train (val) 데이터 fit 결과 — 진짜 calibration 이 아닌 memorization
- Test Brier 미변화 (0.191 → 0.195, 약간 악화) → 일반화 X
- **AUC 0.60 수준 모델에서는 calibration 이 모델 한계를 노출시킴, 개선시키지 X**

### D. Sample 수 부족
- val=626, test=464 sample 으로 isotonic 의 piecewise constant 가 stable fit 어려움
- 이론적 추천: ≥1000 sample 또는 cross-validated calibration (`CalibratedClassifierCV`)

## 7. 배운 점

1. **AUC 보존 보장 ≠ Test AUC 유지** — IsotonicRegression 의 monotonic 보장은 train data (val) 의 ranking 만. test 의 raw proba 가 train 범위 외로 나가면 clip → ranking 변경 가능. 학술 보장과 실제 production 결과의 차이.
2. **Step function calibration 의 plateau 부작용** — piecewise constant fit 이 quantile 기반 threshold 와 충돌. q70 과 q90 collapse → 의사결정 단계 사라짐. **calibration 과 threshold 는 함께 디자인되어야**.
3. **Val Brier 개선 ≠ 실제 calibration** — single-set fit Brier 는 memorization 효과. test Brier 에서 미변화 → 일반화 X. **out-of-sample Brier 측정 필수**.
4. **모델 한계 vs calibration** — AUC ~0.60 모델에서 calibration 은 noise 만 더 잘 calibrate 할 뿐. **신호 자체가 약하면 calibration 도 의미 없음**. B-4 (새 데이터) 가 우선순위.
5. **B-1 rollback 패턴 재사용 가치** — config flag 로 코드 보존 + 실험 재현 가능. 새 데이터 또는 변형 (Platt, CV) 후 즉시 재시도 가능. **fail-safe 패턴의 누적 가치**.
6. **Sprint 의 의사결정 규칙 정교화 필요** — "둘 중 하나라도 baseline 이상" 규칙은 표면적, 실용 의사결정은 (1) test 우선, (2) 부수효과 (threshold collapse) 검증 필요. 다음 sprint 의사결정 규칙은 더 구체화.
7. **하루 5 sprint 의 한계** — 같은 날 5번째 sprint, 작은 scope 임에도 의외 부작용 (threshold collapse) 발견. 신규 변경마다 부수효과 점검의 가치.
