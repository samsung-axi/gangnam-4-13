
### 평가

- **Test AUC 0.6139** — 첫 0.6 돌파. random (0.5) 대비 +0.114 lift. random split 인플레 (0.85) 와는 여전히 거리이지만, 정직한 시계열 holdout 에서 의미 있는 개선
- **Test 의 모든 metric 개선** — AUC, PR-AUC, P@10, R@10 (Brier 동일) — 일관된 신호 향상
- **Val AUC 하락** — random fluctuation 가능성. val 분기 (2023Q1~Q4) 가 test 분기 (2024Q1~Q3) 보다 더 가까운 train data 분기 (2022 까지) 에 있어 inner-join 으로 sample 수 줄면서 추정 분산 ↑
- **B-1 vs A-3 의 차이** — B-1 은 feature 확장만 (학습 신호 직접 추가), A-3 는 alignment + hierarchy (구조적 변경). 후자가 더 안정적

## 5. 배운 점

1. **Inner-join alignment 의 의외의 효과** — val 에서 2건만 mismatch 였지만 그 영향이 metric 측정값에 노이즈로 작용 가능. ensemble 의 기본 가정 (same key) 을 명시적으로 강제하는 것은 학술 정직성의 기본
2. **Hierarchy 의 진짜 가치** — B-1 (단순 feature 확장) 은 실패했지만 A-2 (구조적 hierarchy) 는 효과 있었음. industry-level dynamic 을 prior 로 도입하는 것은 broadcast (B-1 cpi/holiday 처럼) 와 다르게 dynamic. **macro-level signal 도 잘 설계하면 의미 있다**.
3. **Test set 신뢰성 우선** — val AUC 하락에도 keep 한 이유: test 가 더 미래, 사용자 의사결정의 ground truth. val 은 hyperparam tuning 용. **test 가 정직한 평가**.
4. **Subagent merge 충돌 발견 못 함** — T2 subagent 가 작성한 `predict_industry_prior` 코드는 단위 test 통과 (synthetic data 에 industry_prior_pred 컬럼 없음). production 에서 LGBM_FEATURES 의 0-fill effect 로 충돌 발견. **단위 test 의 한계** — production-realistic data 로 integration test 추가 필요
5. **"안 B with safety" 패턴의 가치** — B-1 에서 학습한 rollback 패턴 (의사결정 규칙 자동화) 으로 A-3 도 안전하게 진행. **fail-safe 가 도전을 가능하게 함**
6. **Two-stage stacking 의 학술적 정당성** — Wolpert (1992) 의 stacked generalization 은 현대 ML 의 표준. 본 sprint 가 학계 패턴을 production model 에 적용한 사례
7. **AUC 0.6 의 의미** — random (0.5) 와 random_split inflation (0.85) 의 사이에서, **정직한 시계열 holdout 에서 0.6** 은 production-ready 의 첫 신호. P@10 = 0.326 = baseline (0.32) 대비 1.02× → 의사결정 보조용으로는 활용 가능, 단독 차단 결정에는 부족
