> - **백엔드 API**(Application Programming Interface. 화면이 서버에 "이 정보 줘" 라고 부탁하는 통신 창구): 화면 뒤에서 데이터 처리·저장·계산을 담당하는 서버 부분.
> - **DB**(Database. 데이터를 저장하는 거대한 표 모음): 사용자·매출·인구 같은 정보가 보관된 창고.
> - **배포**: 만든 코드를 실제 서비스 서버에 올려서 사용자가 쓸 수 있게 하는 작업.
> - **ML**(Machine Learning. 머신러닝 — 데이터로 패턴을 학습해 예측하는 모델): 폐업 위험·매출 예측 등에 쓰이는 통계 모델.
> - **DL**(Deep Learning. 딥러닝 — 신경망을 여러 층 쌓아 복잡한 패턴을 학습): 인구·매출 시계열 예측에 사용.
> - **LLM**(Large Language Model. 챗GPT·Claude 같은 대형 언어 모델): 문장을 이해하고 생성하는 AI.
> - **RAG**(Retrieval-Augmented Generation. 외부 문서를 검색해 참고하며 답변하는 AI 기법): 법률 조항을 찾아 답하는 식.
> - **ABM**(Agent-Based Model. 에이전트 기반 시뮬레이션 — 가상의 사람 1000 명을 컴퓨터 안에서 살게 해서 상권 변화를 관찰하는 모델): 마포구 1000 명 가상 주민 시뮬.

### 1-2. 즉시 결정이 필요한 5 가지

1. **운영 RDS 비밀번호 회전(주기적으로 새 비밀번호로 바꾸는 일)** — `models/closure_risk/data_prep.py:165` 등 5 곳에 RDS 호스트(서버 주소)·계정·비밀번호가 평문으로 박혀 있습니다. 깃 히스토리(코드를 누가 언제 어떻게 바꿨는지 모두 저장해 둔 변경 기록부)에도 남아 있어 즉시 비밀번호를 바꾸고 fallback 자체를 제거해야 합니다.
2. **5 개 무인증 관리자 엔드포인트 패치** — 다른 회원의 매니저(다른 사용자를 관리할 수 있는 중간 관리자 역할)를 승인·거절·강제 탈퇴시킬 수 있는 API 가 인증(이 사람이 진짜 그 계정 주인인지 확인하는 절차) 없이 열려 있습니다. URL 만 알면 누구나 접근 가능합니다.
3. **프런트엔드 빌드 정상화** — 현재 프로덕션(실제 사용자에게 서비스되는 운영 환경) 빌드는 타입 검사를 건너뛰고 있어, 타입 오류가 있어도 그대로 배포됩니다.
4. **prod docker-compose(여러 서버 컨테이너를 한 번에 띄워 주는 설정 파일) 에 마이그레이션(DB 구조를 새 버전으로 바꾸는 작업) 명령 추가** — 스키마(DB 테이블 구조 설계도) 변경 배포 시 수동 누락 가능성 상존.
5. **법률 학교 mock fallback 안전판** — DB 미연동 상태에서 마포 전체 학교를 5 개로 판정 → 학교 인근 주점 출점을 "안전" 으로 잘못 안내할 수 있습니다.

## 2. 무엇이 문제인지 — 비전문가 설명

엔지니어가 아니어도 이해할 수 있도록 핵심 위험을 비유로 풀어 썼습니다. 자세한 내용은 도메인별 파일 참조.

### 🔴 보안 위험 — "문이 잠겨 있지 않다"

- **운영 DB 비밀번호 평문 노출** — 회사 금고 비밀번호가 사무실 게시판에 적혀 있는 상태입니다. 외부 침입자가 코드 한 번만 보면 운영 데이터 전체에 접근 가능합니다.
- **인증 없는 관리자 API** — 누구라도 URL 만 알면 다른 회원을 탈퇴시키거나 매니저를 승인·거절할 수 있습니다.
- **권한 우회 가능** — 비밀번호 변경 시 "나는 master(시스템 최고 관리자) 야" 라고 클라이언트(사용자 화면 쪽)가 주장하면 그대로 믿어 줍니다. 일반 직원이 사장 계정 비밀번호를 바꿀 수 있는 구조입니다.

### 🔴 무결성 위험 — "결과가 틀려도 모른다"

- **무음 실패 패턴 만연** — 페르소나 주입·SHAP(SHapley Additive exPlanations. AI 가 왜 그렇게 판단했는지 변수별 기여도를 알려 주는 설명 기법) 설명·DB 조회 등이 실패해도 로그(시스템이 무슨 일을 했는지 시간순으로 적어 두는 기록)가 남지 않고 빈 결과로 폴백합니다. 운영 중 결과가 틀렸을 때 원인 파악이 거의 불가능합니다.
- **법률 mock fallback** — DB 가 연결되지 않을 때 "마포구에 학교는 5 개" 라고 가정합니다. 실제로는 수십 개. 학교 옆 주점 출점을 "안전" 으로 안내할 수 있습니다.
- **ABM 정책 다양성 축소** — 설계상 55 개의 고유 LLM 정책을 만들도록 되어 있지만, 함수 인자(함수에 전달하는 입력값) 누락으로 실제로는 11 개를 5 번 복사하는 동작입니다. 기획 의도와 실제 동작이 다른 채로 배포됨.

### 🔴 운영 위험 — "배포 시 깨질 수 있다"

- **프로덕션 빌드 타입 검사 우회** — 한국식 비유로는 "도장 찍기 전 검토 단계 건너뛰는 것" 입니다. 타입 오류가 있어도 그대로 배포됩니다.
- **prod 마이그레이션 명령 누락** — 스키마 변경 배포 시 수동 작업이 빠지면 화면이 500 에러(서버가 처리 중 터졌다는 표준 오류 코드)로 가득해집니다.
- **Redis 외부 노출** — 캐시 서버가 인터넷에 그대로 열려 있습니다. 인증도 없습니다.

### 🟡 품질 위험 — "기술 부채(지금 빨리 끝내려고 대충 짠 코드가 나중에 갚아야 할 빚으로 쌓이는 것) 누적"

- **프런트엔드 `as any` 38 회** — TypeScript(자바스크립트에 타입 검사 기능을 더한 언어) 의 안전망을 38 곳에서 끄고 있습니다(`as any` 는 "이 값의 타입을 검사하지 마" 라고 강제 지정하는 코드). 점진 제거 필요.
- **백엔드 `except Exception: pass` 다수** — 에러를 잡아서 그대로 무시(`except Exception: pass` 는 "어떤 오류가 나든 그냥 넘어가" 라는 뜻의 파이썬 코드). 운영 가시성 0.
- **DB ORM 관계 정의 0 개** — 44 개 외래키(FK — Foreign Key. 한 테이블이 다른 테이블의 어떤 행을 가리키는지 연결해 주는 열) 관계가 모두 raw SQL(ORM 의 도움 없이 직접 손으로 쓴 SQL 명령문) JOIN(두 테이블을 연결해서 함께 조회하는 SQL 연산). 변경 시 안전망 없음.
- **딥러닝 시드 미고정** — TCN 모델이 매번 다르게 학습됨. 결과 재현 불가능.

## 5. 강점 — 유지·확장 권장 패턴

리뷰 과정에서 발견된 모범 사례입니다. 향후 다른 영역도 동일 수준으로 끌어올리면 좋겠습니다.

- **`simulationStore` 설계** — Promise.allSettled(여러 비동기 작업을 다 기다리되, 일부가 실패해도 전체가 함께 죽지 않게 해 주는 자바스크립트 함수) + monotonic guard(시간순으로 단조 증가하는 ID 같은 걸 두어 늦게 도착한 옛날 응답이 새 응답을 덮어쓰지 못하게 막는 보호 장치) + partialize(상태의 일부만 골라 저장하는 기능). 좀비 상태(끝났는데 안 사라지고 유령처럼 남아 있는 데이터) 방지·재시작 안전성 모두 갖춤.
- **Multi-stage Dockerfile(Docker 이미지를 단계별로 나눠 빌드해 최종 이미지에는 필요한 것만 남기는 기법) + BuildKit pip 캐시(Docker 의 빠른 빌드 엔진과 파이썬 패키지 캐시 결합)** — backend / frontend 양쪽 모두 이미지(컨테이너 실행에 쓰는 파일 묶음) 경량화.
- **dev/prod compose 분리** — prod 에서 코드 volume mount(호스트 폴더를 컨테이너 안에 그대로 연결해 주는 기능) 제거로 격리성 확보.
- **CI 변경 파일 감지 선택적 빌드** + `concurrency.cancel-in-progress`(같은 작업이 중복 실행될 때 진행 중인 옛 작업을 자동 취소하는 GitHub Actions 설정) — 파이프라인(여러 자동화 단계를 줄줄이 이어 둔 흐름) 효율 우수.
- **pre-commit hook**(코드를 커밋하기 직전 자동으로 검사·정리해 주는 자동 실행 스크립트) — Prettier(코드 자동 정렬 도구) + Ruff(파이썬 코드 검사·정리 도구) 자동 포맷·재스테이징(고친 결과를 다시 커밋 대기 상태로 올림).
- **Temporal split(시간 기준으로 학습/검증 데이터를 나누는 방법 — 미래 데이터가 학습에 새지 않도록) + leakage(학습할 때 보면 안 될 미래 정보가 새어 들어가서 모델 점수가 부풀려지는 현상) 정량화** — ML 4.4%p 측정 사례. 학술 인용 stacked generalization(여러 모델의 예측을 또 다른 모델로 결합하는 기법, Wolpert 1992).
- **MNAR(Missing Not At Random. 데이터가 무작위가 아니라 어떤 이유로 빠져 있는 경우 — 단순 평균값으로 채우면 편향이 생김) 기록 문화** — 음수 결과(시도해 봤는데 효과가 없었던 결과)를 명시적으로 문서화.
- **4-layer cache(자주 쓰는 데이터를 4 단계로 나눠 보관해 점점 빠르게 꺼내는 캐시 구조)** — cold start(처음 켤 때 캐시가 비어서 느린 상태) -46%.
- **chunk_compressor(긴 문서를 작은 토막으로 나눠 LLM 에 넣기 전 압축하는 모듈) `<<<CHUNK>>>` prompt-injection(악의적인 사용자가 LLM 입력에 숨긴 명령어로 AI 를 속여 원래 시스템 지시를 무시하게 만드는 공격) 방어**.
