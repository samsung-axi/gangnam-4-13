## 🎯 지표별 상세 가이드

### 1️⃣ WAPE — 주 판정 지표

#### 정의
```
WAPE = Σ|실제−예측| / Σ|실제| × 100%
```
**"전체 매출 규모 대비 예측 오차가 몇 %인가"**

#### 왜 주 지표로 채택했나
- MAPE와 달리 작은 셀에서 **폭발하지 않음** (분모가 전체 합)
- 규모 가중 (큰 셀 오차가 비례적으로 반영)
- M5 Walmart 경쟁 공식 지표

#### 예시
```
실제값:   [100만, 1억, 5억]
예측값:   [80만, 1.2억, 4.8억]
절대오차: [20만, 2천만, 2천만]

WAPE = (20만 + 2천만 + 2천만) / (100만 + 1억 + 5억)
     = 4,020만 / 6억 100만
     = 6.7%
```

#### Production 기준
| WAPE | 판정 | 해석 |
|:--:|:--:|:-----|
| <10% | ⭐ Excellent | World-class 예측 |
| <15% | ✅ Production | 실무 배포 가능 |
| 15~25% | ⚠️ Marginal | 신뢰구간 명시 필수 |
| >25% | 🚫 Not Ready | 재학습 필요 |

**근거:** Simpson & Tranmer (2005) IPF SAE 수용 범위 8~18%

---

### 2️⃣ Pearson r — 선형 관계 강도

#### 정의
```
r = Σ[(xᵢ−x̄)(yᵢ−ȳ)] / √[Σ(xᵢ−x̄)² · Σ(yᵢ−ȳ)²]
```
**"예측이 실제와 같은 방향·기울기로 움직이는가"**

#### 주의점 ⚠️
**r이 높아도 스케일 오차는 포착 못함.**
- 실제 [1, 2, 3], 예측 [10, 20, 30] → r=1.0 (완벽)
- 하지만 예측값이 실제의 10배! → WAPE=900%

**→ WAPE와 반드시 병행**

#### 우리 r = 0.965 의미
- 마포 매출 순위·방향 **거의 완벽히 맞춤**
- "동별 랭킹"이나 "매출 성장 방향" 예측에 신뢰 가능

#### Production 기준
| r | 판정 |
|:--:|:--:|
| >0.95 | ⭐ Excellent |
| >0.90 | ✅ Production |
| 0.80~0.90 | ⚠️ Marginal |
| <0.80 | 🚫 Not Ready |

---

### 3️⃣ R² — 분산 설명력

#### 정의
```
R² = 1 − Σ(y−ŷ)² / Σ(y−ȳ)²
```
**"모델이 실제값 변동의 몇 %를 설명하는가"**

#### Pearson r과의 차이
- **Pearson r**: 방향·순위만 본다 (스케일 무시)
- **R²**: 스케일·편향·잔차 모두 반영

예: 예측값이 실제값의 2배라면?
- r = 1.0 (선형 관계 완벽)
- R² = 음수 (평균 예측보다 못함)

#### 우리 R² = 0.923 의미
- 매출 변동의 **92.3% 설명**
- 예측이 스케일·편향 모두 잘 맞음

#### Production 기준
| R² | 판정 |
|:--:|:--:|
| >0.90 | ⭐ Excellent |
| >0.80 | ✅ Production |
| 0.60~0.80 | ⚠️ Marginal |
| <0.60 | 🚫 Not Ready |

---

### 4️⃣ RMSLE — Log 공간 오차 (초소형 셀 강건)

#### 정의
```
RMSLE = √[mean((log(1+실제) − log(1+예측))²)]
```
**"log 공간에서의 오차 = 배수 오차"**

#### 왜 중요한가
- 작은 매출(0 가까이)에서 MAPE는 폭발 → RMSLE는 안정
- `exp(RMSLE) = 평균 배수 오차`

#### 우리 RMSLE = 0.327 의미
- `exp(0.327) = 1.386`
- **"평균적으로 실제의 1.4배 이내 규모로 예측"**
- 예측이 실제의 0.72배 ~ 1.39배 범위

#### 예시 해석
| RMSLE | 배수 오차 | 해석 |
|:--:|:--:|:--:|
| 0.0 | 1.00 | 완벽 |
| 0.1 | 1.11 | 약 10% 오차 |
| 0.2 | 1.22 | 약 20% 오차 |
| **0.327** | **1.39** | **약 40% 배수 오차** |
| 0.5 | 1.65 | 약 65% 오차 |
| 1.0 | 2.72 | **2~3배 오차** |

#### Production 기준
| RMSLE | 판정 |
|:--:|:--:|
| <0.2 | ⭐ Excellent (Kaggle top 10%) |
| <0.3 | ✅ Production |
| 0.3~0.5 | ⚠️ Marginal |
| >0.5 | 🚫 Not Ready |

**근거:** Kaggle 소매 예측 경쟁(Rossmann, Walmart) 우승 모델 0.15~0.35 구간

### 5️⃣ MASE — Naive 대비 상대 정확도 (Hyndman 원조)

#### 정의
```
MASE = MAE / MAE_naive
```
**"우리 모델이 단순 평균 예측(naive)보다 몇 배 나은가"**

#### 의미
- MASE = 1.0 → naive와 동일 (쓸 이유 없음)
- **MASE < 1.0 → naive보다 우수 (배포 최소 요건)**
- MASE = 0.5 → naive 대비 2배 정확
- MASE = 0.224 → **naive 대비 4.5배 정확**

#### 우리 값 해석
- "만약 모든 셀을 전체 평균으로만 예측했다면 MAE = X"
- 우리 모델 MAE = 0.224 × X
- **정보 가치가 확실히 있음** 증명

#### Production 기준
| MASE | 판정 |
|:--:|:--:|
| <0.5 | ⭐ Excellent |
| <1.0 | ✅ Production |
| 1.0~2.0 | ⚠️ Marginal |
| >2.0 | 🚫 Not Ready (naive가 나음) |

**근거:** Hyndman & Koehler (2006) — "MAPE 대체 표준 지표"

### 6️⃣ F1 (4-tier) vs F1 (binary) — ⚠️ 혼동 주의

연속형 regression을 **분류 문제로 재해석**한 지표. 두 종류가 있고 **해석 완전 다름**.

#### F1 (4-tier) = 0.838 — "세밀 등급 분류"

매출을 사분위수로 4등급 분류:
```
L  (Low)       : 0 ~ p25 (하위 25%)
ML (Mid-Low)   : p25 ~ p50
MH (Mid-High)  : p50 ~ p75
H  (High)      : p75 ~ (상위 25%)
```

**판단:** "예측값의 등급이 실제 등급과 같은가?"

**평균 방식:** Macro F1 = 4개 클래스 F1 평균

#### F1 (binary) = 0.928 — "이분 분류"

매출을 중앙값(median=2.59억) 기준 2개로 분류:
```
0 (Low)  : <2.59억
1 (High) : ≥2.59억
```

**판단:** "중앙값 기준 올바른 쪽에 있는가?"

#### 비교 예시

| 실제 | 예측 | 실제 tier | 예측 tier | 4-tier 정답? | Binary 정답? |
|:--:|:--:|:--:|:--:|:--:|:--:|
| 0.8억 | 0.9억 | L | L | ✅ | ✅ |
| 0.8억 | 1.5억 | L | ML | ❌ | ✅ (둘 다 <2.59억) |
| 2.3억 | 3.0억 | ML | MH | ❌ | ❌ (경계 넘음) |
| 2.3억 | 2.5억 | ML | ML | ✅ | ✅ |
| 5.0억 | 10억 | H | H | ✅ | ✅ |

#### 왜 4-tier > binary는 보통 성립?
- 경계가 3개(4-tier) vs 1개(binary)
- 예측 오차가 경계를 넘을 확률 더 높음
- **Binary가 더 쉬운 문제** → 점수도 더 높음 (0.928 vs 0.838)

#### 비즈니스 사용 시나리오

| 상황 | 추천 F1 | 이유 |
|:-----|:--:|:-----|
| "우수 상권 자동 선별" | Binary (0.928) | 93% 정확 — 상/하 구분만 필요 |
| "투자 우선순위 H→L 리스트" | 4-tier (0.838) | 84% 정확 — 세밀한 등급 필요 |
| "상권 등급 리포트" | 둘 다 병기 | 용도 구분 |

#### Production 기준
| F1 (4-tier) | 판정 | F1 (binary) | 판정 |
|:--:|:--:|:--:|:--:|
| >0.85 | ⭐ | >0.95 | ⭐ |
| >0.75 | ✅ | >0.85 | ✅ |
| 0.60~0.75 | ⚠️ | 0.70~0.85 | ⚠️ |
| <0.60 | 🚫 | <0.70 | 🚫 |

### 7️⃣ OoM (Order-of-Magnitude) — 배수 오차 누적

#### 정의
```
OoM_X = ( |log10(예측/실제)| ≤ log10(X) 인 셀의 비율 )
```

#### 자주 쓰는 구간
| 임계 | log10(임계) | 의미 |
|:--:|:--:|:--:|
| 1.3배 이내 | 0.11 | 실제의 0.77배 ~ 1.30배 |
| **2배 이내** ⭐ | 0.30 | 실제의 0.50배 ~ 2.0배 |
| 3.2배 이내 | 0.50 | 실제의 0.31배 ~ 3.2배 |
| 10배 이내 | 1.00 | 자릿수 맞음 |

#### 우리 결과
| 임계 | 비율 |
|:--:|:--:|
| 1.3배 이내 | 71.4% |
| **2배 이내** | **95.7%** ⭐ |
| 3.2배 이내 | 98.1% |
| 10배 이내 | 99.8% |

→ **"95.7%의 예측이 실제의 2배 범위 안"** → 자릿수 맞추는 작업엔 매우 신뢰

#### Production 기준 (OoM 2배 이내)
| 비율 | 판정 |
|:--:|:--:|
| >95% | ⭐ Excellent |
| >90% | ✅ Production |
| 80~90% | ⚠️ Marginal |
| <80% | 🚫 Not Ready |

### 8️⃣ MAPE — 참고용만

#### 정의
```
MAPE = mean(|실제−예측| / |실제|) × 100%
```

#### 경고 ⚠️
**본 task에 부적합.** 이유:
- 실제값이 작을 때 분모가 작아져 **폭발** (예: 실제 100만 → 50만 오차 → 50% MAPE)
- 본 프로젝트 초기 v1 실험에서 MAPE=135% 기록 (Lewis 경고의 전형)

#### Lewis (1982) 4단계 스케일 (전통적)
| MAPE | 판정 |
|:--:|:--:|
| <10% | Highly Accurate |
| 10~20% | Reasonable |
| 20~50% | Inaccurate |
| >50% | Unusable |

#### 우리 판단
**MAPE 25.5%는 Lewis "Inaccurate" 이지만, 이는 소규모 셀 편향 탓.**
→ 참고용으로만 보고, WAPE/RMSLE/MASE를 주 지표로 사용.

### 🔟 MAE — 원 단위 절대 오차

#### 정의
```
MAE = mean(|실제 − 예측|)
```

#### 우리 값
- **median: 2,900만원**, mean: 5,630만원
- → "**평균적으로 ±3,000만원 틀린다**"

#### 용도
- 비즈니스 설명용 ("이 예측은 평균 ±X원 오차")
- 스케일 비교엔 부적합 (WAPE/RMSLE 사용)

## 🏁 종합 판정 규칙

### 복합 판정 공식
```
Production-Ready =
  WAPE < 15%           ← 핵심
  AND Pearson r > 0.90 ← 방향성
  AND (R² > 0.80 OR RMSLE < 0.3) ← 스케일 정확성 OR 안정성
  AND MASE < 1.0       ← naive 이김
  AND OoM 2배 > 90%    ← 자릿수 정확
```

### 우리 ExtraTrees v3 판정
- ✅ WAPE 13.35% < 15%
- ⭐ r 0.965 > 0.90
- ⭐ R² 0.923 > 0.80 (AND RMSLE 0.327 ≈ 0.3)
- ⭐ MASE 0.224 < 1.0
- ⭐ OoM 95.7% > 90%

**→ 모든 조건 충족, Production-Ready 확정** ✅
