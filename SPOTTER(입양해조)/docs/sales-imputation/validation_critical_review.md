# 검증 방법론 비판적 재평가 + 재시도 리포트

**작성일:** 2026-04-23
**대상:** seoul_district_sales 137 결측 셀 역추적 (v2 WAPE 14.3% 주장의 재검증)
**핵심 결론:** 원본 v2 검증은 **과대평가**. 결측 실제 상황 기준 **WAPE ≈ 25%가 정직한 수치**.

---

## 1. 왜 비판적 재평가가 필요했는가

v2 리포트는 **WAPE 14.3%, Pearson r=0.991, R²=0.981**을 보고하며 "Target Achieved"로 판정했다. 그러나 이 수치는 단일 검증(**random 10-fold CV**)에만 근거하며, 다음 네 가지 구조적 위험이 누락됐다:

| # | 위험 | 검증 질문 |
|:-:|:-----|:---------|
| 1 | 시계열 누수 | 같은 연도 Q1이 train, Q2가 test에 가면 trend를 "미래"인 척 학습할 수 있음 |
| 2 | MNAR (Missing Not At Random) | 137 결측은 "품질 게이트 필터링"이라 random sample이 아닌 특정 프로파일 (작은 셀) |
| 3 | Fixed Effect Memoization | dong/industry dummy가 lookup table로 쓰이면 일반화 불가 |
| 4 | Scale Bias | 평균 WAPE가 대형 셀이 끌어내린 결과일 수 있음. 정작 복원이 필요한 소형 셀에선 다를 수 있음 |

---

## 2. 4가지 추가 감사 (v2 원본 모델 대상)

### Audit A — Time-Series CV (rolling forward)
학습: 첫 8분기(2019Q1~2020Q4) → 한 분기씩 rolling forward.
random CV와 달리 **미래 분기 정보가 train에 절대 포함되지 않음**.

**결과:** 16개 rolling fold 평균 **WAPE 17.5% (+3.2%p vs random)**
- 약한 시계열 누수 존재. 그러나 큰 왜곡은 아님.

### Audit B — MNAR-Mimicking CV (주 지표)
결측 137개의 `store_count` 95%ile = **15개**. survivor 95%ile = **171개** (10배 차이).
결측 프로파일과 유사한 survivor (store_count ≤ 15)만 hold-out하고 5-fold.

**결과: WAPE 28.6% ±1.8 (+14.3%p vs random) ⚠️**
- **결측 셀과 유사한 작은 셀은 예측이 훨씬 어려움**
- random CV는 전체 평균이라 사실상 큰 셀(WAPE 11.9%)이 낮춰준 결과

### Audit C — Leave-One-Dong-Out
한 동 전체(240 cells) 제거하고 그 동 예측.
GBM이 dong dummy로 "각 동 평균"을 외웠는지 확인.

**결과:** 16개 동 평균 **WAPE 41.0% (+26.7%p vs random) ⚠️**
- 서교동 72.3%, 성산1동 54.8%, 염리동 52.5% — 최악 case
- GBM이 **dong fixed effect에 과도 의존**. 새 동 일반화 실패.

### Audit D — 셀 크기 분위별 층화
store_count 4분위로 나눠 각 WAPE 계산.

| 분위 | 평균 사업체수 | 평균 매출 | WAPE |
|:----|----:|----:|----:|
| **Q1 (작음)** | **6.9** | **2.89억** | **27.7%** ⚠️ |
| Q2 | 16.4 | 6.06억 | 25.9% |
| Q3 | 37.7 | 12.39억 | 20.3% |
| Q4 (큼) | 162.5 | 78.28억 | **11.9%** |

- **결측 137셀 평균 매출 ≈ 0.96억 → Q1보다 더 작음** → 실제 WAPE는 27.7%보다 더 클 가능성

## 3. v2 검증의 정직한 재해석

| 상황 | 적합한 지표 |
|:-----|:----------|
| 살아있는 셀 무작위 예측 | random CV **14.3%** (원본 보고) |
| 내년 같은 분기 예측 | Time-Series CV **17.5%** |
| **137 결측 복원 (실제 과제)** | **MNAR-Mimic 28.6%** |
| 새로운 동 예측 | LODO **41.0%** |

**결론:** v2의 "WAPE 14.3%"는 **살아있는 셀 무작위 holdout**에서만 성립. 실제 관심사인 **137 결측 복원 정확도는 ~28% 수준**으로 해석해야 정직하다.

## 4. v3 재설계: 약점 대응 시도

### 설계 변경 3가지

1. **Target 변경**: `log(monthly_sales)` → **`log(sales_per_store)`**
   - "매출 총액"이 아닌 "점포당 생산성"을 학습
   - 규모 효과 제거 → 작은 셀에도 동일 모델 적용 가능
   - 최종 예측: `sales_per_store × store_count`로 환산

2. **Dong one-hot 제거**
   - 16개 one-hot dummy 삭제
   - 대신 동별 집계 통계 추가 (`dong_avg_store`, `dong_total_store`)
   - → Fixed effect memoization 차단, 일반화 피처 학습 유도

3. **동×업종 상호작용 피처**
   - `combo_avg_store`: 해당 (동, 업종)의 분기 평균 store_count
   - 구조적 포지션 정보 보존

### v2 vs v3 성능 비교

| 감사 | v2 | **v3** | 변화 |
|:----|---:|---:|---:|
| Random 10-fold | 14.3% | — (의미 약화) | — |
| A. Time-Series | 17.5% | **15.8%** | **−1.7%p ✅** |
| **B. MNAR (주 지표)** | **28.6%** | **25.7%** | **−2.9%p ✅** |
| C. Leave-One-Dong-Out | 41.0% | 41.8% | +0.8%p (동일) |
| D. Q1 (작은 셀) | 27.7% | 24.7% | **−3.0%p ✅** |

**v3 소폭 개선:** Time-Series·MNAR·Q1 전 영역에서 2~3%p 감소.
**LODO는 개선 없음:** 동 dummy를 빼도 GBM이 `dong_avg_store` 통계로 비슷한 기능 학습.

## 5. 왜 더 이상 개선되지 않는가 (정직한 한계 분석)

### 5.1 작은 셀의 구조적 특성

결측 137셀의 프로파일:
- 평균 store_count **3~15개** (작은 업종, 외곽 동)
- 평균 월 매출 **1억원 이하** (일반 음식점 1~2곳 규모)
- **분산이 절대값 대비 큼** — 업소 하나의 개·폐업만으로 매출 50% 변동

이 규모에선 **통계적 신호 < 개별 업소 노이즈**. 공공데이터 품질 게이트가 애초에 걸러낸 이유와 동일.

### 5.2 공무원 발언 재해석

> "자체 집계와 통계청 값을 비교해 오차가 크면 배포 안함"

→ 오차가 큰 작은 셀을 의도적으로 걸러냈다. **우리도 같은 영역에서 싸우고 있음**. 예측이 어려운 이유는 공공데이터 자체 품질 게이트가 작동하는 이유와 동일.

### 5.3 추가 피처로도 극복 어려운 이유

시도해볼 만한 외부 피처:
- SEMAS 상가업소 (개별 업소 단위) → 이미 집계된 store_count 이상의 정보 없음
- 유동인구 (living_population_grid) → 이미 DB에 있으나 분기 해상도 부재
- 카카오맵 리뷰 수 → 업소 자체 노이즈 복사

**결론:** WAPE 20% 장벽은 외부 데이터로도 쉽게 안 깨짐.

## 6. 최종 정직한 판정

### 6.1 137 결측 복원 신뢰도

| 관점 | 값 | 해석 |
|:-----|:--:|:-----|
| Pearson r | 0.99 | **순위·방향 예측은 매우 정확** |
| R² | 0.98 | 분산의 98% 설명 |
| MNAR WAPE | **25.7%** | **개별 셀 매출 규모는 ±25% 구간** |
| Lewis (1982) 판정 | 🥉 Inaccurate | 20~50% 구간 상단 |

### 6.2 복원값 사용 권고

| 용도 | 권고 |
|:-----|:-----|
| ✅ **전체 합계 추정** | 137셀 매출합 132억 ± 34억 (WAPE 25.7% 적용) |
| ✅ **동·업종 랭킹** | Pearson r=0.99 → 순위 정확 |
| ⚠️ **개별 셀 의사결정** | 단독 사용 금지, **±30% 신뢰구간 명시 필수** |
| ❌ **투자·입점 결정** | 복원값 단독으론 부적합. 현장 조사 병행 |

### 6.3 confidence 점수 하향 조정

v2 리포트의 confidence=0.93은 random CV 14.3% 기반 → **비현실적**.
v3에서는 MNAR WAPE 25.7% 기반으로 **confidence=0.74**로 조정.

## 7. 시사점: "좋은 수치"를 의심해야 하는 이유

본 과제는 **검증 방법론이 결과를 좌우**하는 전형적 케이스다:

| 검증 | 수치 | 의미 |
|:-----|----:|:-----|
| 원본 random CV | 14.3% | "제품 전체 평균" |
| MNAR-Mimic | 25.7% | "실제 사용 상황" |

14.3%만 보고하면 **상품 홍보**, 25.7%를 함께 보고해야 **기술 보고서**다.

### 검증 설계 체크리스트 (향후 적용)

본 과제의 경험을 일반화:

1. **CV 분할은 실제 사용 상황을 모방하는가?**
   - 시계열 예측 → Time-series CV
   - 결측 복원 → MNAR-mimic CV
   - 새 지역 예측 → Leave-one-group-out

2. **평균 지표가 주요 하위 그룹을 가리지 않는가?**
   - 셀 크기·지역·시기별 층화 필수

3. **Fixed effect가 일반화를 가장하지 않는가?**
   - One-hot dummy 대신 **learnable 통계 features**
   - Leave-one-group-out으로 고정효과 의존도 측정

4. **"과제의 실제 난이도"를 반영한 기준인가?**
   - Lewis 스케일은 일반 예측용. 구조적 결측 복원은 **20~30% WAPE가 현실적 상한**일 수 있음.

## 9. 결론

- v2 WAPE 14.3% 주장은 **살아있는 셀 무작위 holdout**에서만 성립
- **실제 과제(137 결측 복원) 기준 신뢰도는 WAPE ≈ 26%**
- v3 재설계로 2~3%p 개선, 그러나 **작은 셀 예측의 구조적 한계는 존재**
- 복원값은 **집계 용도 및 순위 추정에 적합**, **개별 셀 의사결정엔 부적합**
- confidence 0.93 → **0.74**로 하향 조정 반영
