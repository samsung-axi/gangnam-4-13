# 발표 시연용 ABM 시뮬레이션 캐시 전략

작성: A1 (찬영) — 2026-04-21

## 1. 문제

발표 중 버튼 클릭 → 실시간 시뮬은 대기 시간 때문에 불가능.

| 모드 | 인구 | 모델 | 소요 시간 |
|---|---|---|---|
| mini | 100명 × MC 3회 | Qwen 로컬 | 5~10분 |
| full | 1000명 | Qwen 로컬 | 약 9시간 |

발표 중 10분 로딩은 시연 불가.

## 2. 두 가지 선택지

### A. 사전 캐시 방식

- 발표 전날 밤에 시연용 시나리오 N개를 미리 시뮬 → JSON 파일로 저장
- 발표 당일 버튼 클릭 → 캐시 파일 즉시 반환

### B. Mock 결과 주입

- 시뮬 없이 미리 만들어둔 가짜 숫자 반환
- 어떤 입력에도 즉시 응답하지만 진짜 데이터 아님

## 3. 추천: 하이브리드 (캐시 + Mock fallback)

```
사용자 클릭
   ↓
캐시 hit? ─── Yes → 실제 시뮬 결과 즉시 반환
   │
   No
   ↓
Mock 결과 반환 (안전망)
```

### 왜 캐시가 우선인가

| 항목 | 사전 캐시 | Mock |
|---|---|---|
| 응답 속도 | 즉시 | 즉시 |
| 데이터 진위 | 실제 시뮬 결과 | 가짜 숫자 |
| 심사 질문 대응 | 가능 (trajectory·스토리·에이전트 동선까지 보여줌) | 불가 |
| 즉흥 입력 | 미스 가능 | 항상 응답 |

심사위원이 "이 매출이 왜 이렇게 나왔어요?"라고 물을 때, 캐시는 실제 시뮬 로그가 살아있어서 답변 가능. Mock은 짜놓은 숫자라 설명 불가.

### 안전망으로 Mock 추가

캐시에 없는 즉흥 조합이 들어와도 빈 화면 대신 mock 응답 → 발표 깨지지 않음.

## 4. 시연 시나리오 후보 (캐시 대상 30개)

| 카테고리 | 조합 예시 |
|---|---|
| 기본 | 평일 / 주말 / 공휴일 |
| 날씨 | 맑음 / 비 / 눈 |
| 충격 | 임대료 +0% / +15% / +30% / +50% |
| 동 특화 | 홍대(서교) / 상암 / 공덕 / 망원 |
| 시간대 | 점심 피크 / 저녁 피크 / 야간 |

조합으로 총 ≈ 30개. 한 시뮬당 5~10분 → 밤새 5~6시간이면 완료.

## 5. 작업 분담

| 담당 | 작업 | 예상 시간 |
|---|---|---|
| B1 | `/api/simulate-abm` 엔드포인트에 캐시 hit 로직 + mock fallback 추가 | 30분 |
| A1 | `scripts/precompute_demo_cache.py` 배치 작성 + 발표 전날 밤 시뮬 실행 | 1시간 + 야간 5~6시간 |

## 6. 캐시 파일 구조 (제안)

```
data/processed/demo_cache/
├── manifest.json                    # 시나리오 키 → 파일명 매핑
├── weekday_clear_rent0.json
├── weekday_rain_rent30.json
├── weekend_clear_rent0.json
└── ... (총 30개)
```

`manifest.json` 키 형식: `{weekday|weekend}_{weather}_rent{pct}_{focus_dong}`

## 7. 결정 필요 사항

1. 추천대로 **하이브리드**로 갈지, 아니면 캐시 단독 / mock 단독으로 갈지
2. 시연 시나리오 30개 최종 확정 (B1·B2·A1·A2 합의 필요)
3. 발표 전날 시뮬 실행 시점 (밤 10시 시작 → 새벽 4시 완료 가정)
