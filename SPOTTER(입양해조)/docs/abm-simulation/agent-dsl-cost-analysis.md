# 마포 ABM · 전원 원시어 DSL 전환 비용 분석

> 현재 Tier S/A/B 계층화 구조 대신 **1,000명 전원 원시어 DSL**로 의사결정하는 경우의 비용·트레이드오프·구현 로드맵.

---

## 1. 제안 개요

**현재**: Tier S(50) + Tier A(200) 만 LLM 호출, Tier B(750)는 파이썬 규칙
**제안**: 1,000명 **전원 LLM 호출**, 단 모든 입출력을 **원시어 DSL**로 극도 압축

→ "1,000명 전체가 LLM 기반으로 움직인다"는 발표 임팩트를 얻으면서도, 토큰 절감으로 오히려 **현재보다 75% 저렴**.

## 3. 토큰 비교

### 3.1 현재 Tier A 프롬프트 (~175 tok)

```
마포 resident 28세, 연남동, 14시 평일. 가성비0.3, 카페선호0.9.
예산잔여 75,000원. 행동 JSON: {"action":"visit|move|rest",
"category":"카페|음식점|편의점|주점|null","spend":원,"reason":"한문장"}
```

### 3.2 DSL 버전 (~28 tok)

**요청 포맷** (파이프 구분자)
```
45|28F|연남|14|75k|c0.9|p0.3
```
| 필드 | 값 | 의미 |
|------|-----|------|
| 45 | agent_id | |
| 28F | 연령+성별 | 28세 여성 |
| 연남 | current_dong | |
| 14 | hour | 14시 |
| 75k | budget_remaining | 75,000원 |
| c0.9 | pref_cafe | 카페 선호 0.9 |
| p0.3 | price_sensitivity | 가성비 성향 0.3 |

**응답 포맷** (단일 DSL verb)
```
VISIT cafe
```
또는
```
MOVE 합정
REST
```

→ **83% 토큰 절감**

## 6. DSL 스펙 (의사결정 Verb)

기존 대화 DSL(`[INV]`, `[ACC]`, `[DEC]`, `[PROMO]`, `[INFO]`)에 **5가지 의사결정 Verb** 추가:

| Verb | 문법 | 의미 |
|------|------|------|
| `VISIT` | `VISIT cafe` / `VISIT food` / `VISIT pub` / `VISIT cvs` | 현재 동에서 카테고리 매장 방문 |
| `MOVE` | `MOVE 합정` | 목적지 동으로 이동 |
| `REST` | `REST` | 휴식 (집/현재 위치) |
| `WORK` | `WORK` | 점주 — 본인 가게 근무 |
| `EAT` | `EAT cafe slow` | 특정 매장 이름 부분매칭 (고급, 선택) |

파서는 현재 `conversation.py`의 `parse_dsl` 확장.

## 7. 구현 로드맵

### Phase 1 — 기반 (1시간)

1. `brain.py`에 `dsl_decide(agent, world) -> Decision` 추가
   - 프롬프트: 파이프 구분 20토큰
   - 응답: VISIT/MOVE/REST/WORK 파싱
   - 실패 시 Tier B 규칙으로 fallback
2. `agents.py`의 `decide()`에 Tier 분기 대신 DSL 모드 토글

### Phase 2 — 프롬프트 최적화 (30분)

3. 에이전트 압축 태그 생성 함수
   - `creative|30F|연남|cafe:hi|budget:mid` 형태
   - 아키타입 + profile 속성 조합
4. System message (공통, 프롬프트 캐싱)
   ```
   Reply one of: VISIT {cafe|food|pub|cvs} | MOVE {dong} | REST | WORK
   ```

### Phase 3 — Narrator (1시간)

5. `narrator.py` — 시뮬 종료 후 DSL 로그 → 자연어 요약
6. `runner.py` 마지막에 1회 호출
7. 결과를 `sim_{name}_narration.json`으로 저장

### Phase 4 — 프론트엔드 (30분)

8. Narrator 결과 fetch + Macro 뷰에 오버레이 표시
9. 테스트 프론트엔드 `/api/simulation/{name}/narration` 엔드포인트 추가

**전체 작업량 약 3시간.**
