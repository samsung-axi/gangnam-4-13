## 오늘 한 것

### 1. 법률 드로어 전면 개선

**이전**: 법률 조문 원문이 수천 자씩 그대로 표시 → 읽기 어려움, 조문 잘림, 쓰레기 텍스트 혼입

**현재**:
- **창업 체크리스트** (상단) — LLM이 업종/지역에 맞춰 생성 (bullet point + 위반 제재)
- **핵심 조항** (하단) — 조문 제목 + 한 줄 요약으로 압축

### 2. RAG 검색 안정화

| 문제 | 해결 |
|------|------|
| RAG 검색 실패 (RDS 커넥션 풀 고갈) | 13개 동시 검색 → 2배치 분할 |
| 실패 결과가 캐시되어 반복 | articles 3개 미만이면 캐시 안 함 |
| "RAG 근거 조항 없음" | zoning/ftc 결과를 articles로 변환 |
| Windows ProactorEventLoop 에러 | --reload 모드(SelectorEventLoop) 필수 확인 |

### 3. RAG 검색 파이프라인 고도화

기존 벡터 검색만 → **하이브리드 검색**으로 업그레이드:

```
쿼리 → HyDE 동의어 매핑 → 벡터 검색 + BM25 검색 → RRF 결합 → 결과
```

- **HyDE**: "4대보험"→"국민연금 건강보험 고용보험 산재보험" 등 21개 일상 용어 자동 치환
- **BM25**: 메모리 기반 키워드 검색으로 벡터가 놓치는 조문번호/고유용어 보충
- **RRF**: Reciprocal Rank Fusion으로 두 결과를 합산 정렬

## 벤치마크 결과

### 검색 정확도

| 평가 방식 | 정확도 | 설명 |
|----------|--------|------|
| Exact-match | **55%** (18/33) | "정확히 그 조문"만 정답 (엄격) |
| **LLM-as-judge** | **72%** (23/32) | "관련 있는 조문"이면 정답 (실질) |

> 55%와 72%의 차이: 반환된 조문이 "틀린" 게 아니라 "다른 관련 조문"이기 때문.
> 예) 식품위생법 제41조(위생교육) 대신 제51조(영업허가 취소) 반환 → 둘 다 창업자에게 필요한 조문.

### 검색 응답 시간

- 평균: **42ms** (첫 쿼리 제외)
- 첫 쿼리: ~10초 (임베딩 모델 로딩, 이후 메모리 캐시)

## 시도했지만 효과 없었던 것

| 시도 | 결과 | 판단 |
|------|------|------|
| 쿼리를 법률 용어로 교체 | 55% → 30% (↓) | 임베딩 모델이 자연어에 최적화 |
| 쿼리 확장 (동의어 추가) | 55% → 42% (↓) | 쿼리가 길어지면 벡터 분산 |
| ko-sroberta 임베딩 모델 교체 | 55% → 21% (↓) | 한국어 특화지만 법률에는 비효과 |
| Cross-encoder Reranker | 55% → 52% (↓) + 16초 지연 | 한국어 법률에 비최적화 |

> 결론: 현재 MiniLM + 원래 쿼리 + HyDE 동의어 + BM25 조합이 최적.
