# A2 법률 에이전트 벤치마크 & 통합 검증 리포트

> 최종 측정일: 2026-04-23 | 담당: 봉환 (A2)

## 1. RAG 검색 정확도

### 1-1. 평가 방법론

#### 평가 지표
- **F1-score**: Precision(정확도)과 Recall(재현율)의 조화평균. RAG 검색 성능의 정량적 측정에 사용
- **LLM-as-judge**: 반환된 조문의 실질 관련성을 GPT-4.1-mini가 판정 (YES/NO)
- **Exact-match**: 기대 조문이 반환 결과에 포함되는지 여부 (엄격 기준)

#### Golden Dataset (정답지) v3
- **구성**: 11개 법률 × 2~3개 주제별 쿼리 = 29개 쿼리, 총 63개 기대 조문
- **선별 기준**: 법률 원문에서 프랜차이즈 창업자가 준수해야 할 의무/허가/신고/금지 조문을 독립 선별
- **쿼리-정답 매칭**: 각 쿼리가 묻는 주제 범위에 해당하는 조문만 정답으로 지정
- **조문 정규화**: 항 분할 접미사(`제2조_0` → `제2조`) 제거하여 조 단위 매칭

> v1~v3 정답지는 법률당 1개 쿼리 × 3개 조문(33개)으로 구성되었으나, 쿼리 범위와 정답 범위 불일치 문제가 있었음. v3(현재)에서 쿼리별 정답 분리로 해결.

#### Golden Dataset 방법론 전환 (v3 → v4)

| 항목 | v3 정답지 | v4 정답지 (현재) |
|------|-----------|------------------|
| 생성 주체 | LLM(Claude) 임의 생성 | 창업 실무 법률 리서치 후 조문 매핑 |
| 선별 절차 | 쿼리에 맞춰 추정 | 법률 원문 목차 → 창업자 의무/금지 조항 식별 |
| 정답지 크기 | 11쿼리 × 3조문 = 33 | 29쿼리 × 2~3조문 = 63 |
| 쿼리-정답 매칭 | 광범위 쿼리에 광범위 정답 | 주제별 쿼리에 주제별 정답 한정 |
| 검증 가능성 | 불가 (LLM 출력 기반) | 법령 원문 대조 가능 |

> v3 → v4로 Exact-match 수치가 63.6% → 34.9%로 하락한 것은 RAG 성능 저하가 아니라 **평가 기준 자체가 엄격해진 결과**.
> v3 정답지는 Claude가 임의 생성한 것으로 지표 자체에 신뢰성 문제가 있었음. v4는 창업 실무 법률을 먼저 조사한 뒤 해당 조문을 정답으로 매핑하여 **도메인 기반으로 검증 가능한 정답지**를 구성.
> 따라서 v3 지표와 v4 지표는 동일 축 비교가 불가능하며, v4가 실제 서비스 품질의 정확한 측정치.

### 1-4. LLM-as-judge 실질 관련성

반환된 조문이 쿼리 주제와 직접 관련 있는지를 GPT-4.1-mini가 판정합니다.

**실질 관련성: 106/121 (87.6%)**

> F1(0.272)과 LLM-judge(87.6%)의 차이 해석:
> - F1이 낮은 이유: 정답지의 **특정 조문**을 정확히 못 찾아서 (Precision 부족)
> - LLM-judge가 높은 이유: 기대 조문 대신 **다른 관련 조문**을 반환하기 때문
>
> 예: "가맹금 반환" 쿼리에 정답이 `제10조`인데 RAG가 `제13조`(허위과장 금지)를 반환
> → F1에선 오답이지만 LLM-judge에선 "관련 있음"으로 판정
>
> **창업자 관점에서는 LLM-judge 87.6%가 실질적인 서비스 품질 지표.**
> 특정 조문 번호를 정확히 찾는 것보다, 관련 법률 내용을 넓게 커버하는 것이 더 중요.
>
> LLM-judge 판정은 GPT-4.1-mini 단일 모델 사용. 5회 반복 측정 시 표준편차 σ=2.77 수준으로 비결정성에 의한 정상 분산 범위.

#### LLM-judge 정확도 검증 (Known-answer Test)

LLM-judge의 안정성(σ=2.77)과 별개로, **판정 자체의 정확도**를 검증합니다.

검증 방법: 확실한 YES 10개 + 확실한 NO 10개 = 20개 Known-answer 케이스를 LLM-judge에 투입하여 예측 정확도를 측정.

| 지표 | 관대한 프롬프트 (v3) | 엄격한 프롬프트 (v4, 현재) |
|------|:------------------:|:----------------------:|
| Accuracy | 55% | **75%** |
| True Positive Rate | 90% | **100%** |
| False Positive Rate | **80%** | **50%** |

**프롬프트 개선 내용:**
- few-shot 예시 4개 추가 (YES 1개 + NO 3개)
- "같은 법률이라도 다른 주제면 NO" 규칙 명시
- "조문 내용을 확실히 알지 못하면 NO (추측 금지)" 규칙 추가

**해석:** FPR 50%는 LLM이 NO인 조문도 절반을 YES로 판정한다는 뜻. 따라서 LLM-judge 87.6%는 **실제 관련성보다 다소 과대평가**일 수 있음. 실제 관련성은 약 70~80% 수준으로 추정.

### 1-7. 검색 파이프라인 아키텍처

4단계 하이브리드 RAG 파이프라인:

```
쿼리 → [0] HyDE 동의어 확장 (70개 법률 용어 사전)
     → [1] 벡터 유사도 검색 (pgvector, paraphrase-multilingual-MiniLM-L12-v2)
     → [2] BM25 키워드 검색 (메모리 역인덱스)
     → [3] RRF 결합 (벡터 50% + BM25 50%)
     → 최종 top_k 반환
     → [4] 벌칙 조문 자동 매핑 (의무 조문 → 과태료/벌금 원문 첨부)
```

| 구성요소 | v3 (4/22) | v4 (4/23, 현재) |
|---------|-----------|----------------|
| Chunking | 조 단위 + 500자 고정 분할 | **조 → 항(①②③) → 문장 의미론적 분할** + `[제N조]` 접두사 |
| 청크 수 | 3,663개 | **5,272개** |
| HyDE 사전 | 37개 | **70개** (법률별 핵심 용어 보강) |
| 조문 정규화 | 없음 | **항 접미사 제거** (`제2조_0` → `제2조`) — 벤치마크 전용, 에이전트는 metadata 원본 사용 |
| 벌칙 매핑 | 없음 | **28개 의무→벌칙 조문 매핑** (원문 자동 첨부) |
| 평가 지표 | Exact-match + LLM-judge | **F1-score** + Exact-match + LLM-judge |
| 정답지 | 11쿼리 × 3조문 = 33개 | **29쿼리 × 2~3조문 = 63개** (주제별 분리) |

## 2. 벌칙 조문 자동 매핑 (v4 신규)

RAG가 의무 조문을 찾으면 연결된 벌칙/과태료 조문 본문을 chunks.json에서 자동 조회하여 recommendation에 첨부합니다.

### 동작 흐름

```
RAG: 식품위생법 제37조 (영업신고 의무) 검색됨
  → PENALTY_ARTICLE_MAP: 제37조 → 제97조
  → chunks.json에서 제97조 본문 조회
  → recommendation에 자동 추가:
    "⚖️ 위반 시 제재: (제97조) 3년 이하의 징역 또는 3천만원 이하의 벌금"
```
