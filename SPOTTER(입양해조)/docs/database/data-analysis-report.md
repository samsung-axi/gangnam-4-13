## 2. 데이터 품질 이슈

### 2.1 행정동 vs 상권 데이터 불일치

서울시 상권분석서비스는 두 가지 단위로 데이터를 제공한다:

- **행정동 단위**: 추정매출, 점포수 → `district_sales`, `store_quarterly`
- **상권 단위**: 골목상권/발달상권/전통시장 → `golmok_*` 테이블

두 데이터는 별도 시스템에서 집계되기 때문에 불일치가 발생한다.

**사례**: 염리동 일식음식점
- `store_quarterly` (행정동 단위): 24분기, 점포 7~12개
- `golmok_stores` (상권 단위): 1분기, 점포 0개
- 원인: 염리동에 매핑된 골목상권(3110581)에 일식음식점이 없음

**조치 완료**: 점포-행정동 원본 CSV 6개 파일(2019~2024)로 `district_stores.csv` 및 `store_quarterly` 교체 → 33,876행으로 보강 후 음식점 10개 업종 필터링(3,840행)

## 4. 딥러닝 학습 전략 (B2 수지니 참고)

### 4.1 권장 방식: 사전학습 + 파인튜닝

결측값을 비율로 채워넣는 방식 대신, **서울 전체 데이터로 사전학습 후 마포구로 파인튜닝**하는 전략을 권장한다.

```
1단계 — 사전학습: 서울 전체 25개구 데이터로 LSTM 학습
                  (매출 + 점포수 + 유동인구)
                  → 업종 공통의 일반 패턴 학습
                    (계절성, 코로나 영향, 경쟁 구조 등)

2단계 — 파인튜닝: 마포구 데이터로 추가 학습
                  (+ 임대료, SGIS 인구 등 마포구 전용 피처)
                  → 마포구 고유 특성 반영

3단계 — 백테스팅: 2024년 마포구 실제 매출과 예측값 비교
```

### 4.2 이 방식의 장점

**결측값 문제 해결**:
- 망원2동 일식(매출 0분기)도 서울 전체 238개 동의 일식 패턴으로 학습 완료
- 마포구 파인튜닝 시 유동인구·점포수 등 외부 피처로 추론 가능
- 비율로 가짜 데이터를 넣는 것보다 근거가 풍부함 (238개 동 vs 비율 1개)

**데이터 확보 용이**:
- 바탕화면 CSV가 이미 서울 전체 25개구 데이터 포함 (마포구 필터링만 해제하면 됨)
- 추정매출: 연간 약 68,000행 × 6년
- 점포수: 연간 약 33,000행 × 6년
- 유동인구: LOCAL_PEOPLE_DONG 파일 84개 (2019.01~2026.02)

### 4.3 결측값 처리 기준

| 구분 | 조합 | 처리 |
|------|------|------|
| 전분기 누락 (2건) | 망원2동 일식, 아현동 양식 | 마포구 파인튜닝에서 **제외**, 사전학습 패턴 + 외부 피처로 추론 |
| 절반 미만 (3건) | 망원2동 양식, 신수동 일식, 성산2동 양식 | **제외** |
| 절반 이상 (6건) | 아현동 패스트푸드 외 5건 | **선형 보간** 후 학습 포함 |
| 24분기 완전 (147건) | 나머지 전부 | 그대로 사용 |

### 4.4 사전학습 모델(Chronos 등) 검토 결과

| 모델 | 외부 피처 | 결측 보완 | 판단 |
|------|----------|----------|------|
| Chronos (Amazon) | ❌ 단변량만 | ❌ 불가 | 베이스라인 비교용으로만 활용 |
| TimesFM (Google) | ❌ 단변량만 | ❌ 불가 | 동일 |
| Lag-Llama | △ 제한적 | ❌ 불가 | 동일 |
| **직접 LSTM (사전학습+파인튜닝)** | **✅** | **✅** | **권장** |

범용 사전학습 모델은 유동인구 같은 외부 피처를 활용할 수 없어, 우리 프로젝트에서는 직접 LSTM을 사전학습+파인튜닝하는 것이 가장 적합하다. 단, Chronos로 빠르게 베이스라인 정확도를 뽑아두면 직접 LSTM의 성능 비교 기준으로 유용하다.

## 5. B1/B2 데이터 사용 가이드

### 행정동 vs 상권 데이터 활용

| 역할 | 사용 데이터 | 용도 |
|------|-----------|------|
| B2 (LSTM 예측) | 행정동 단위 (`district_sales`, `store_quarterly`) | 분기별 매출 시계열 학습 |
| B1 (Agent 시뮬레이션) | 행정동 + 상권 둘 다 | 행정동: 큰 그림 / 상권: 개별 입지 상세 분석 |

### B1 시뮬레이션 시 데이터 흐름

사용자가 "망원1동에 치킨집 출점" 요청 시:
1. `district_sales` → 해당 동의 업종 매출 추세
2. `store_quarterly` → 경쟁 점포수, 폐업률
3. `dong_mapping`의 `trdar_codes` → 해당 동에 속한 상권 코드 조회
4. `golmok_stores` → 어떤 상권에 경쟁점이 밀집해있는지
5. `golmok_sales` → 상권별 매출 차이 분석
