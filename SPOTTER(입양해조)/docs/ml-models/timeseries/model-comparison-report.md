## 1. 비교 개요

### 1.1 비교 목적

마포구 프랜차이즈 상권분석 시뮬레이터의 매출 예측 기능에 사용할 최적 모델을 선정하기 위해
LSTM, GRU, TCN 세 가지 시계열 예측 모델을 동일 조건에서 구현하고 비교한다.

세 모델은 모두 마포구 분기별 매출 데이터(dong_code × industry_code 조합)를 기반으로
2024년 연간 매출을 예측하며, 실제 2024년 매출과의 오차를 측정하여 성능을 평가한다.

### 1.2 동일 조건 요약

세 모델이 공정하게 비교될 수 있도록 아래 조건을 완전히 통일하였다.

| 조건 | 공통값 | 비고 |

## 2. 아키텍처 비교

### 2.1 구조 개요

| 항목 | LSTM | GRU | TCN |
|------|------|-----|-----|
| **구조 유형** | 순환 신경망 (RNN) | 순환 신경망 (RNN) | 팽창 인과 컨볼루션 |
| **게이트 수** | 3개 (input, forget, output) | 2개 (reset, update) | 없음 |
| **hidden state** | h + c (cell state) | h만 | 없음 |
| **병렬 처리** | 불가 (시퀀셜) | 불가 (시퀀셜) | 가능 |
| **기울기 소실 완화** | cell state (forget gate) | update gate | Residual connection |
| **수용 영역 제어** | hidden state 간접 | hidden state 간접 | dilations=[1,2]로 직접 (RF=4) |
| **시퀀스 집약** | Attention (가중 합산) | Attention (가중 합산) | 마지막 타임스텝 `[:,:,-1]` |
| **파라미터 수** | 약 229K (최다) | 약 168K | 약 145K (최소) |
| **epoch당 학습 시간** | ~8초 | ~7.8초 | ~25초 |

### 2.2 hidden state 구조 차이

```python
# LSTM: 두 가지 내부 상태 반환 (hidden state + cell state)
lstm_out, (h, c) = self.lstm(x)   # h: 단기, c: 장기 기억 분리

# GRU: hidden state만 반환 (cell state 없음)
gru_out, _ = self.gru(x)          # h만으로 단/장기 기억 통합 처리

# TCN: 순환 구조 없음 — 입력을 한 번에 병렬 처리
x = self.input_proj(x)            # Linear 투영 (31 → 128)
x = x.transpose(1, 2)             # (batch, seq, ch) → (batch, ch, seq)
for block in self.tcn_blocks:
    x = block(x)                  # TemporalBlock 순차 통과
out = x[:, :, -1]                 # 마지막 타임스텝만 사용
```

LSTM의 cell state는 forget gate를 통해 장기 패턴(예: 코로나 전후 구조 변화)을 명시적으로 보존한다.
GRU는 cell state 없이도 update gate로 유사한 역할을 수행하며 파라미터가 적다.
TCN은 hidden state 자체가 없고, 팽창 컨볼루션의 수용 영역(RF=4)으로 시퀀스 전체를 한 번에 처리한다.

### 2.3 hidden_size vs n_channels 차이

| 모델 | 파라미터명 | 값 | 의미 |
|------|-----------|-----|------|
| LSTM | `hidden_size` | 128 | RNN 내부 상태 벡터 차원 수 |
| GRU | `hidden_size` | 128 | RNN 내부 상태 벡터 차원 수 |
| TCN | `n_channels` | 128 | 각 TemporalBlock의 Conv1d 채널 수 |

숫자는 128로 동일하지만 개념이 다르다.
`hidden_size`는 순환 연산에서 매 타임스텝 전달되는 상태 벡터의 크기이고,
`n_channels`는 컨볼루션 필터의 개수(출력 채널 수)이다.
두 값을 동일하게 설정하여 모델 용량(capacity)을 최대한 통일하고 공정한 비교를 도모했다.

### 2.4 아키텍처 흐름 비교

**LSTM / GRU (공통 구조):**
```
입력 (batch, 4, 31)
  → nn.LSTM / nn.GRU (hidden=128, layers=2)
  → Attention: Linear(128→64) → Tanh → Linear(64→1) → Softmax
  → context (batch, 128)
  → FC: Linear(128→64) → ReLU → Dropout → Linear(64→1)
  → 출력 (batch, 1)
```

**TCN (다른 구조):**
```
입력 (batch, 4, 31)
  → input_proj: Linear(31→128)
  → transpose → (batch, 128, 4)
  → TemporalBlock(dilation=1): CausalConv1d × 2 + LayerNorm + Residual
  → TemporalBlock(dilation=2): CausalConv1d × 2 + LayerNorm + Residual
  → 마지막 타임스텝 [:, :, -1] → (batch, 128)
  → FC: Linear(128→64) → ReLU → Dropout → Linear(64→1)
  → 출력 (batch, 1)
```

## 3. 학습 결과 비교

### 3.1 사전학습 (서울 전체 25개구, 87,938행)

| 항목 | LSTM | GRU | TCN |
|------|------|-----|-----|
| 설정 epoch | 100 | 100 | 100 |
| 실제 종료 epoch | **42** (조기종료) | 65 (조기종료) | 58 (조기종료) |
| **best val_loss** | 0.000336 | 0.000318 | **0.000287** |
| epoch당 시간 | ~8초 | ~7.8초 | ~25초 |
| 가중치 경로 | `pretrained.pt` | `pretrained_gru.pt` | `pretrained_tcn.pt` |

- TCN이 사전학습 val_loss 최저 — 전체 서울 데이터 피팅 성능은 TCN 우수
- TCN은 epoch당 약 3배 느림 (input_proj + 2개 TemporalBlock + LayerNorm transpose 오버헤드)
- LSTM이 가장 일찍 종료 (epoch 42) — Attention 레이어 추가로 인한 약한 과적합 경향

### 3.2 파인튜닝 1단계 (마포구, 3,703행 — RNN/TCN freeze, FC만 학습)

| 항목 | LSTM | GRU | TCN |
|------|------|-----|-----|
| 설정 epoch | 10 | 10 | 10 |
| 실제 종료 epoch | 10 (전체 완료) | 10 (전체 완료) | 10 (전체 완료) |
| **최종 val_loss** | 0.000503 | 0.000484 | 0.000441 |
| 학습률 | 5e-4 | 5e-4 | 5e-4 |

- TCN이 1단계에서도 val_loss 최저 — FC Head가 마포구 스케일에 빠르게 적응
- LSTM이 val_loss 가장 높음 — 파라미터 수가 많아 FC만으로는 충분히 적응하지 못함

### 3.3 파인튜닝 2단계 (전체 unfreeze, 낮은 학습률)

| 항목 | LSTM | GRU | TCN |
|------|------|-----|-----|
| 설정 epoch | 50 | 50 | 50 |
| 실제 종료 epoch | 27 (조기종료) | **13** (조기종료) | **12** (조기종료) |
| **best val_loss** | 0.000479 | 0.000462 | 0.000420 |
| 학습률 | 1e-4 | 1e-4 | 1e-4 |
| 가중치 경로 | `finetuned_mapo.pt` | `finetuned_mapo_gru.pt` | `finetuned_mapo_tcn.pt` |

- TCN이 파인튜닝 val_loss도 최저 (0.000420) — 손실 함수 기준 최우수
- GRU·TCN은 13·12 epoch에서 조기종료 — 소규모 마포구 데이터에서 빠르게 수렴
- LSTM은 27 epoch까지 지속 개선 — 파라미터가 많아 세밀한 조정 기간 필요

## 4-4. 종합 분석

### 1. 랜덤 3회 vs 시드 고정 3회 일관성 분석

두 조건(시드 미설정 / 시드 고정) 모두에서 LSTM이 MAPE 기준 1위를 유지했다.

| 조건 | LSTM MAPE | GRU MAPE | TCN MAPE | 1위 |
|------|-----------|----------|----------|-----|
| 랜덤 3회 (실험 A) | **10.44%** | 11.38% | 11.61% | **LSTM** |
| 시드 고정 3회 (실험 B) | **10.88%** | 11.45% | 11.43% | **LSTM** |
| 전체 6회 | **10.66%** | 11.41% | 11.52% | **LSTM** |

LSTM은 학습 조건과 무관하게 상대 오차(MAPE) 기준 일관된 우위를 보였다. 이는 단순 운이 아닌 실제 모델 성능 차이로 해석할 수 있다.

### 2. 모델별 안정성 분석 (MAPE 표준편차 기준)

| 모델 | 랜덤 3회 표준편차 | 시드 고정 3회 표준편차 | 전체 6회 표준편차 | 안정성 평가 |
|------|---------------|-------------------|---------------|-----------|
| LSTM | 0.15% | 0.61% | 0.54% | 중간 |
| GRU  | 0.22% | 0.80% | 0.64% | 가장 불안정 |
| TCN  | 0.90% | **0.11%** | 0.59% | 시드 고정 시 가장 안정 |

- **TCN**: 시드 고정 조건에서 표준편차 0.11%p — 세 모델 중 시드에 가장 안정적. 랜덤 조건의 0.90%p와 대조적으로 시드 고정 효과가 가장 두드러짐.
- **GRU**: 시드 고정 조건에서도 0.80%p로 가장 큰 변동성 — 동일 시드여도 내부 연산 특성상 결과 편차 발생, 시드에 가장 민감한 구조.
- **LSTM**: 랜덤 조건(0.15%)에서는 가장 안정적이나, 시드 고정(0.61%)에서는 중간 수준 — 전체 6회 기준 0.54%로 균형 잡힌 안정성.

### 3. 최종 모델 선정: LSTM

**선정 근거:**

1. **6회 일관된 1위 유지**: 랜덤 3회 + 시드 고정 3회, 총 6회 모든 실험에서 MAPE 기준 최우수. 단 한 회차도 GRU·TCN에 역전되지 않음.
2. **전체 6회 평균 MAPE 최저**: LSTM 10.66% vs GRU 11.41% vs TCN 11.52% — 약 0.75~0.86%p 차이.
3. **실험 조건 독립적 우위 확인**: 시드 설정 여부와 무관하게 동일한 순위 유지 → 모델 구조 자체의 실질적 성능 우위로 확인.
4. **균형 잡힌 전반적 성능**: MAE·RMSE·R²에서 TCN이 절대 오차 기준 우세하나, MAPE(상대 오차) 기준에서는 LSTM이 일관되게 우수 — 마포구 동·업종별 매출 규모 편차가 큰 환경에서 상대 오차 지표가 더 적합한 평가 기준임.

## 5. 종합 분석

### 5.1 모델별 특성 요약

#### LSTM

| 항목 | 내용 |
|------|------|
| **예측 방식** | 3-gate 순환 신경망 + Attention 가중 합산 → 자기회귀 4스텝 |
| **장점** | cell state로 장기 패턴 보존 / Attention으로 중요 분기 강조 / 전 동·업종 균형 예측 |
| **단점** | 파라미터 최다(~229K) → 소규모 데이터에서 과적합 위험 / 학습 속도 느림 |
| **한계** | 마포구 156개 조합의 소규모 데이터에서는 파라미터 과잉 가능성 / 1회 학습 결과 |
| **비고** | MAPE·MAE 최우수 / 전반적으로 가장 균형 잡힌 성능 |

#### GRU

| 항목 | 내용 |
|------|------|
| **예측 방식** | 2-gate 순환 신경망 + Attention 가중 합산 → 자기회귀 4스텝 |
| **장점** | 파라미터 적음(~168K) → 소규모 데이터 과적합 위험 낮음 / LSTM과 유사 구조 |
| **단점** | cell state 없어 장기 의존성 표현력 LSTM보다 약함 / 모든 지표에서 세 모델 중 최하위 또는 중간 |
| **한계** | MAE 14.8억으로 LSTM(10.3억) 대비 크게 높음 — 대형 상권 절대 오차 큼 |
| **비고** | LSTM 대비 MAPE 차이 0.9%p (통계적으로 유의미한 차이인지 반복 실험 필요) |

#### TCN

| 항목 | 내용 |
|------|------|
| **예측 방식** | 팽창 인과 컨볼루션 (RF=4) → 마지막 타임스텝 → 자기회귀 4스텝 |
| **장점** | R²·RMSE 최우수 / 대형 상권 절대 오차 최소 / hidden state 누적 오차 없음 |
| **단점** | MAPE 12.6%로 세 모델 중 최고 / epoch당 학습 시간 3배 느림(~25초) |
| **한계** | RF=4 고정으로 4분기 이상 장기 패턴 반영 불가 / 소규모 불규칙 상권에서 상대 오차 큼 |
| **비고** | 파인튜닝 val_loss 최저(0.000420)지만 백테스트 MAPE는 최고 — 과소 표현 조합에서 취약 |

### 5.3 마포구 데이터 특성 적합도 분석

마포구 데이터는 아래 특성을 갖는다:
- **소규모**: 154개 동×업종 조합, 마포구 분기 데이터 3,703행
- **이질적**: 상암동(대형 업무지구) ↔ 염리동(소규모 주거상권) — 매출 스케일 수백 배 차이
- **구조 변화**: 코로나(2020~2021) 전후 급격한 패턴 변화 존재
- **계절성**: 분기별 패턴이 규칙적인 동(상암동)과 불규칙한 동(신수동) 혼재

| 특성 | LSTM 적합도 | GRU 적합도 | TCN 적합도 |
|------|------------|-----------|-----------|
| 소규모 데이터 | 보통 (파라미터 과잉 가능) | 좋음 (파라미터 적음) | 좋음 (파라미터 최소) |

**결론**: 마포구의 소규모·이질적 데이터 특성에서는 **LSTM이 가장 균형 잡힌 성능**을 보인다.
TCN은 대형 상권(상암동, 서교동)에서 강하지만 소규모 불규칙 상권에서 상대 오차가 커지는 경향이 있다.

## 7. 페르소나 기반 지표 우선순위 재검토

### 7-1. 핵심 페르소나

본 시스템의 1차 타겟 사용자는 **프랜차이즈 본사 입지 개발팀**이다. 이들의 핵심 목적은 마포구 전체 상권 패턴을 파악하고, 신규 출점 후보지를 선별하여 본사 전략 보고서에 활용하는 것이다. 개인 자영업자의 단일 점포 수익 예측이 아닌, 동·업종 조합 전체에 걸친 매출 분포와 패턴 설명력이 핵심이다.

이 관점에서 평가 지표의 중요도는 단순 예측 정확도 순서가 아닌, **의사결정 리스크** 기준으로 재정의된다.

### 7-2. 페르소나 관점 지표 우선순위

| 우선순위 | 지표 | 이유 |
|----------|------|------|
| 1순위 | **R²** | 전체 상권 패턴 설명력 → 전략 보고서의 신뢰성 근거 |
| 2순위 | **RMSE** | 극단 오차 최소화 → 대형 상권 잘못 예측 시 수억 원 규모 손실 |
| 3순위 | **MAE** | 평균 절대 오차 → 입지 분석 보고서 전반의 예측 정확도 |
| 4순위 | **MAPE** | 상대 오차율 → 소규모 상권 기준, 모델 간 차이가 근소함 |

> MAPE는 소규모 상권(연매출 수억 원대)에서 분모 효과로 오차율이 과장될 수 있으며, 대형 상권 중심의 전략적 의사결정에서는 절대 오차 지표(RMSE·MAE)와 전체 설명력(R²)이 더 유효한 판단 기준이다.

### 7-3. 페르소나 기준 모델별 적합도 분석

#### 실험 A — 랜덤 초기화 3회 평균 기준

> 수치 출처: `validation/results/lstm_backtest_results*.csv`, `gru_backtest_results*.csv`, `tcn_backtest_results*.csv` (run1~run3)

| 지표 | LSTM | GRU | TCN | 최우수 |
|------|------|-----|-----|--------|
| **R² (1순위)** | 0.9839 | 0.9770 | **0.9928** | **TCN** |
| **RMSE (2순위)** | 37.3억 | 43.4억 | **24.6억** | **TCN** |
| **MAE (3순위)** | 10.3억 | 12.9억 | **9.5억** | **TCN** |
| **MAPE (4순위)** | **10.44%** | 11.38% | 11.61% | **LSTM** |

- TCN이 1~3순위(R²·RMSE·MAE) 3개 지표 모두 최우수
- LSTM은 4순위(MAPE)에서만 우세
- GRU는 전 지표에서 LSTM·TCN 대비 열위

#### 실험 B — 시드 고정 3회 평균 기준

> 수치 출처: `validation/results/lstm_backtest_results_seed*.csv`, `gru_backtest_results_seed*.csv`, `tcn_backtest_results_seed*.csv` (seed47/415/2026)

| 지표 | LSTM | GRU | TCN | 최우수 |

| **RMSE (2순위)** | 31.0억 | 43.3억 | **23.4억** | **TCN** |
| **MAE (3순위)** | 9.5억 | 12.7억 | **8.7억** | **TCN** |

- MAPE에서 TCN 11.42%, LSTM 10.88%로 0.54%p 차이 — 두 조건 중 가장 근접
- TCN MAPE 표준편차 0.10%p로 3개 모델 중 가장 안정적

## 8. 최종 모델 선정

### 8-1. 선정 모델: TCN

페르소나(프랜차이즈 본사 입지 개발팀) 관점의 지표 우선순위를 적용한 결과, **최종 모델로 TCN을 선정**한다.

**선정 근거:**

1. **R²·MAE·RMSE 3개 지표에서 랜덤/시드 고정 두 조건 모두 일관되게 1위**
   실험 A(랜덤 3회)와 실험 B(시드 고정 3회) 모두에서 TCN이 R²·RMSE·MAE 3개 지표 최우수를 유지했다. 총 6회 실험에서 단 한 회차도 역전되지 않은 일관된 우위다.

2. **시드 고정 시 MAPE 표준편차 0.10%p — 3개 모델 중 가장 안정적**
   시드 고정 조건에서 TCN의 MAPE 표준편차는 0.10%p로, LSTM(0.61%p)·GRU(0.80%p) 대비 월등히 낮다. 동일 시드로 재학습해도 결과가 거의 변하지 않는 구조적 안정성을 의미한다.

3. **프랜차이즈 본사 입지 개발팀 관점에서 전체 상권 패턴 설명력과 극단 오차 최소화가 전략적 의사결정에 더 중요**
   R²=0.9928(랜덤 3회 평균)은 전략 보고서에서 "이 모델이 상권 패턴의 99.3%를 설명한다"는 신뢰 근거로 활용된다. RMSE 24.6억(랜덤 3회 평균)은 대형 상권의 이상 예측으로 인한 수억 원 규모 의사결정 오류를 방지한다.

4. **MAPE는 LSTM이 0.86%p 낮으나 절대 차이 근소하며 타겟 페르소나 기준 우선순위 낮음**
   랜덤 3회 기준 MAPE 차이는 LSTM 10.44% vs TCN 11.61%로 1.17%p이나, 시드 고정 3회 기준으로는 LSTM 10.88% vs TCN 11.42%로 0.54%p까지 줄어든다. 또한 MAPE는 소규모 상권에서 분모 효과로 과장되는 경향이 있어, 대형 상권 중심의 전략적 출점 결정에서는 본 페르소나 기준 4순위 지표로 판단했다.

### 8-2. LSTM과의 비교 분석

| 항목 | LSTM 유리 | TCN 유리 |
|------|-----------|----------|
| **지표** | MAPE (상대 오차율, 4순위) | R²·RMSE·MAE (1~3순위) |
| **적합 상황** | 소규모 상권 상대 오차 최소화, 편의점·카페 등 저매출 업종 분석 | 전체 상권 패턴 설명, 대형 상권 극단 오차 최소화, 전략 보고서 |
| **안정성** | 랜덤 조건 표준편차 0.15%p (가장 낮음) | 시드 고정 조건 표준편차 0.10%p (가장 낮음) |
| **페르소나 적합도** | 소규모 점포 자영업자 관점 | 프랜차이즈 본사 입지 개발팀 관점 |

> **결론**: LSTM은 MAPE 단일 지표에서 우세하며 소규모 상권 상대 오차 최소화에 강점이 있다. 반면 TCN은 본 시스템의 타겟 페르소나인 프랜차이즈 본사가 전략적 의사결정에서 중시하는 전체 상권 설명력(R²), 대형 상권 극단 오차 제어(RMSE), 평균 절대 오차(MAE) 3개 지표에서 일관된 우위를 보인다. 페르소나 기준 지표 우선순위를 적용하면 TCN이 최적 모델이다.

### 8-3. 운영 시 주의사항

- **시드 미설정 환경에서 TCN 불안정**: 랜덤 조건 3회 MAPE 표준편차 0.74%p(run1: 12.61%, run2: 10.85%, run3: 11.38%) — 실험 조건에 따라 최대 1.76%p 편차 발생 가능
- **실제 서비스 배포 시 시드 고정 학습 권장**: `--seed` 파라미터 명시적 지정으로 재현 가능한 학습 환경 구성 필요
- **정기적 재학습 시에도 시드 고정 필요**: 재학습마다 시드를 동일하게 유지해야 결과 일관성 확보 가능
