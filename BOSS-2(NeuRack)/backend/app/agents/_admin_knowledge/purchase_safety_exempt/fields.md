# 구매안전서비스 비적용대상 확인서 — 프로필 필드 매핑

## profiles 컬럼 → 양식 placeholder 매핑

| 양식 placeholder        | profiles 컬럼 | profile_meta 키  | 비고                           |
| ----------------------- | ------------- | ---------------- | ------------------------------ |
| {{representative_name}} | display_name  | —                | 신고인 성명                    |
| {{today}}               | —             | —                | 작성일 (오늘 날짜 자동)        |
| {{exemption_reason}}    | —             | exemption_reason | 소명 내용 (없으면 placeholder) |

## LLM 채움 규칙

1. today는 항상 오늘 날짜(YYYY년 MM월 DD일)로 채울 것
2. exemption_reason: profile_meta에 값이 있으면 채우고, 없으면 `{{해당 사유에 대한 소명 내용 직접 기재}}`로 유지
3. 해당 사유 체크박스는 LLM이 사용자 메시지에서 언급한 사유를 파악해 [x]로 표시
   - 사용자가 "신용카드만 받아요" → 첫 번째 항목 [x]
   - 사용자가 "디지털 상품" / "소프트웨어" → 두 번째 항목 [x]
   - 사용자가 "정기구독" / "나눠서" → 세 번째 항목 [x]
   - 사용자가 "공공기관" / "후불" → 네 번째 항목 [x]
   - 언급 없으면 모두 [ ] 유지 (사용자가 직접 체크)
