# UX/UI Skill 도출 및 디자인 시스템 수립 과정 🎨

본 문서는 "어디다있소" 서비스의 UX/UI 가이드라인(Skill)이 어떤 논리와 근거를 바탕으로 설계되었는지 그 도출 과정을 기록합니다.

## 1. 브랜드 핵심 아이덴티티 추출 (Brand DNA)
다이소(Daiso) 브랜드의 핵심 요소를 디지털 서비스에 녹여내는 것을 최우선 순위로 삼았습니다.

- **핵심 컬러**: 다이소의 상징인 **Red (#E50000)**를 `Primary Color`로 설정하여 사용자에게 브랜드 일관성을 제공합니다.
- **로고 전략**: 서비스명 "어디다있소"에서 "다있소" 부분을 강조하여 브랜드 인지도를 높였습니다. (상단 빨강/하단 검정 그라데이션 기법 적용)
- **비주얼 톤**: 복잡한 장식보다는 **직관적인 면(Solid)**과 **명확한 타이포그래피**를 사용하여 신뢰감 있고 깔끔한 느낌을 구현했습니다.

## 2. 사용자 인터랙션 모델 설계
사용자의 환경(키오스크 vs 모바일)에 따라 다른 최적의 인터랙션을 도출했습니다.

### 2.1 Kiosk (Tablet) 환경
- **터치 Target**: 공공 장소에서 사용되는 만큼 버튼의 최소 크기를 확보하고 곡률(`border-radius: 25px`)을 주어 시각적 안정감을 부여했습니다.
- **음성 중심 UI**: 중앙의 거대한 마이크 버튼과 애니메이션을 통해 키보드 입력보다 음성 검색이 더 편리함을 유도했습니다.
- **정보 전달**: 로딩 중에도 "상품을 찾는 중... 85%"와 같은 상태 메시지와 마스코트를 제공하여 대기 시간을 긍정적인 경험으로 전환했습니다.

### 2.2 Mobile 환경
- **연결성**: 키오스크의 결과(QR)를 통해 즉시 모바일 지도로 전환되는 흐름을 설계했습니다.
- **AR 네비게이션**: 카메라 피드 위에 직관적인 화살표 오버레이를 배치하여 텍스트 위치 가이드보다 명확한 경로 안내를 제공합니다.

## 3. 디자인 시스템 도출 과정 (Rule Setting)
실제 개발 시 일관성을 유지하기 위해 공통 규칙을 세분화했습니다.

1.  **그리드에서 좌표로**: 단순 4x4 그리드는 매장 실제 크기를 반영하지 못하므로, 평면도 이미지 기반의 **% 좌표 핀 방식**을 도출했습니다.
2.  **공통 컴포넌트**: `result-card`와 같은 컴포넌트에 Shadow와 Border 규칙을 정의하여 CSS 코드의 재사용성을 극대화했습니다.
3.  **반응형 전략**: 모바일과 태블릿의 브레이크포인트를 각각 768px, 1024px로 설정하여 레이아웃이 유연하게 변하도록 설계했습니다.

## 4. 결론
이러한 도출 과정을 통해 생성된 `SKILL.md`는 단순히 스타일을 정의하는 것을 넘어, **"다이소 매장을 가장 쉽고 빠르게 안내하는 서비스"**라는 본질에 충실하도록 설계되었습니다.
