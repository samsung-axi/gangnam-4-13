# 화면 구성 (6개 핵심 화면)

## 1. 키오스크 홈 (태블릿)

**구성 요소 (위→아래):**

1. **헤더**: 좌측 다이소 로고 + "어디다있소", 우측 KOR/ENG 토글 + 도움말 아이콘
2. **추천 상품 캐러셀**: 카드형 상품 슬라이더 ("오늘의 추천 상품", 가격 태그)
3. **음성 검색 버튼**: 중앙 큰 빨간 원형 마이크 아이콘 + 파동 애니메이션
4. **검색 안내 텍스트**: "상품 이름을 말씀해주세요"
5. **텍스트 검색 입력창**: 마이크 버튼 하단, 키보드 입력 가능
6. **하단 탭 바**: 카테고리 지도 | 상품 위치 찾기

**인터랙션:**
- 마이크 버튼 탭 → 음성 녹음 시작 (버튼 파동 애니메이션, 녹음 중 상태 표시)
- 음성 입력 완료 또는 텍스트 제출 → Stage 1의 3번(의도분석)으로 전환
- 캐러셀 좌우 스와이프 → 추천 상품 탐색
- 하단 탭 → 해당 화면으로 전환

---

## 2. 로딩 화면 (태블릿)

**진입 조건:** Stage 2의 5번(검색 실행) 시 표시

**구성 요소:**

1. **원형 프로그레스 바**: 빨간색(#FF0000) dashed 스타일, 중앙에 퍼센티지 표시
2. **마스코트 캐릭터**: 빨간 쇼핑백 캐릭터 (프로그레스 바 좌측 하단에 걸침)
3. **상태 텍스트**: "상품을 찾는 중..." (Bold, SUIT font)
4. **검색어 표시**: 말풍선 형태로 사용자가 입력한 키워드 표시 (예: "규조토 욕실 매트")

**전환:** 검색 완료 시 → 3번 화면(검색 결과)으로 자동 전환

---

## 3. 검색 결과 화면 (태블릿)

**진입 조건:** Stage 2의 7번(결과 화면 출력)

**구성 요소 (좌우 분할 레이아웃):**

**좌측 패널 (30%):**
1. "검색 결과 N개" 타이틀
2. 상품 카드 리스트 (선택된 카드 = 빨간 좌측 보더 4px)
   - 각 카드: 상품 썸네일 + 상품명 + 위치코드(📍 B1-B01) + 가격
3. QR 코드 영역 (상품 선택 후 표시)
4. 카카오톡 보내기 버튼 (노란색 #FEE500)

**우측 패널 (70%):**
1. B1 / B2 층 전환 탭
2. 매장 평면도 이미지 (frontend/images/map_b1.png 또는 map_b2.png)
3. 파란 점선 이동 경로 (현재위치 → 상품위치, map-pathfinding.md 데이터 기반)
4. 빨간 핀 마커 — 상품 위치
5. 파란 원 마커 — 현재 위치 (입구)

**인터랙션:**
- 상품 카드 탭 → 우측 지도에 해당 상품 경로 표시 + QR 코드 생성
- B1/B2 탭 전환 → 해당 층 평면도로 변경
- QR 코드 → 모바일 스캔 시 Stage 4 진입

**결과 없음 시:** 좌측 패널에 "검색 결과가 없습니다" + 재검색 버튼 + 인기 검색어 추천

---

## 4. 카테고리 지도 화면 (태블릿)

**진입 조건:** 홈 화면 하단 탭 "카테고리 지도" 선택

**구성 요소:**

**좌/중앙 영역:** B1 Floor + B2 Floor 매장 평면도 (좌우 나란히 또는 탭 전환)

**우측 사이드바:** 카테고리 필터 버튼 목록
- 전체, 뷰티, 주방, 욕실, 문구, 수납정리, 식품, 인테리어, 애견
- 각 버튼: 아이콘 + 텍스트, 선택 시 빨간 배경(#FF0000) + 흰색 텍스트

**인터랙션:**
- 카테고리 버튼 탭 → 해당 카테고리 매대 영역 지도에서 하이라이트
- 하이라이트된 매대 탭 → 해당 카테고리 상품 목록 표시

---

## 5. 모바일 지도 화면 (모바일)

**진입 조건:** Stage 4의 11번(QR 스캔) 후 모바일 웹 오픈, 12-1 선택 시

**구성 요소:**

1. **헤더**: ← 뒤로가기 + "어디다있소" 타이틀
2. **상품 정보 카드**: 상품 이미지 + 상품명 + 위치코드 + "약 N초 소요" (경로 기반 추정)
3. **매장 평면도**: 해당 층 지도 + 경로 점선 + 현재위치(파란 원) + 목적지(빨간 핀)
4. **하단 버튼 2개**:
   - "지도 보기" — 빨간 배경, 현재 활성 상태
   - "AR 네비" — 흰 배경 + 빨간 테두리, 탭 시 6번 화면으로 전환

---

## 6. AR 네비게이션 화면 (모바일)

**진입 조건:** 5번 화면에서 "AR 네비" 버튼 탭, 또는 Stage 4의 12-2 선택 시

**구성 요소:**

1. **헤더**: "어디다있소" + 연결 상태 아이콘 + 메뉴
2. **카메라 뷰**: 전체화면 카메라 피드 (실제 카메라 또는 시뮬레이션 영상)
3. **AR 오버레이**:
   - 3D 빨간 화살표 (진행 방향 지시)
   - 거리 텍스트 ("12m 앞으로")
   - 도착지 마커 핀 + 위치코드 ("B1-B01")
4. **하단 상품 카드**: 상품 이미지 + 상품명 + 남은 거리 + 예상 소요시간
5. **토글 버튼**: "지도 보기" ↔ "AR 네비" 전환

> **참고**: AR 네비게이션은 실내 측위 인프라(BLE 비콘 등)가 없으므로
> MVP에서는 정적 시뮬레이션(고정 방향 화살표 + 거리 텍스트)으로 구현합니다.
> 실시간 위치 추적 기반 AR은 향후 확장 범위입니다.
