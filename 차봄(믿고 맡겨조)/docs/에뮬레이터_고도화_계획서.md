# OBD 에뮬레이터 고도화 계획서

## 1. 목표 및 배경

### 목표
최근 프론트엔드 및 백엔드 블루투스 OBD 시스템에 추가된 신규 기능들을 **PC 환경에서 완전히 시뮬레이션**할 수 있도록 에뮬레이터를 고도화합니다.

### 배경
- 프론트엔드에 **8개의 신규 PID** 지원이 추가됨 (MAP, MAF, 흡기온도, 엔진가동시간 등)
- **DTC 자동 보고** 기능 추가 (Mode 03 조회 후 AI 서버 전송)
- **배치 ID 기반 멱등성** 처리 로직 구현
- **자동 주행 종료** 감지 기능 추가
- 현재 에뮬레이터는 기본 4개 PID만 지원하여 신규 기능 테스트 불가

### 예상 효과
- 실제 차량 없이도 **모든 신규 로직을 100% 테스트** 가능
- 개발-테스트 사이클 단축 (실차 필요 → PC만으로 가능)
- DTC 발생, 네트워크 단절 등 **엣지 케이스 시나리오** 재현 용이
- 개발 효율성 **200% 이상** 향상

## 2. 현재 에뮬레이터 상태 분석

### 지원 현황

#### 현재 지원되는 기능
- ✅ AT 명령어 처리 (ATZ, ATE0, ATRV 등)
- ✅ Mode 01 기본 PID (RPM, Speed, Coolant, Load, Throttle, Voltage)
- ✅ Mode 03 DTC 조회 (기본 로직)
- ✅ Mode 04 DTC 삭제
- ✅ Mode 09 VIN 조회
- ✅ CSV 기반 데이터 재생 (Replay 모드)
- ✅ 다중 차량 프로필 설정 (config.yml)

#### 부족한 부분
- ❌ 신규 PID 미지원 (010B, 0110, 010F, 011F)
- ❌ Mode 02 (Freeze Frame) 미구현
- ❌ Mode 03 DTC 변환 로직 불완전 (10진수 기준으로 변환 중)
- ❌ CSV 매핑 로직이 구조화되지 않음 (PID 기반 매핑 필요)
- ❌ Engine Runtime 증가 로직 없음 (시간 기반 자동 증가 필요)

## 3. 고도화 항목 상세

### 3.1 신규 PID 추가

#### 추가할 PID 목록

| PID | Mode | 이름 | 설명 | 데이터 바이트 | 변환 공식 |
|-----|------|------|------|--------------|----------|
| 010B | 01 | MAP | 흡기 매니폴드 압력 | 1 | A (kPa) |
| 0110 | 01 | MAF | 공기 유량 | 2 | ((A×256)+B) / 100 (g/s) |
| 010F | 01 | Intake Temp | 흡기 온도 | 1 | A - 40 (°C) |
| 011F | 01 | Engine Runtime | 엔진 가동 시간 | 2 | (A×256)+B (초) |

#### 구현 계획

**run_emulator.py 수정**:
```python

self.pids = {
    # ... 기존 PIDs ...
    '010B': '00',          # MAP
    '0110': '00 00',       # MAF
    '010F': '00',          # Intake Temp
    '011F': '00 00',       # Engine Runtime
}

# Engine Runtime 자동 증가 타이머
self.engine_start_time = None
```

**update_pids_from_dict 메서드 확장**:
```python
if data.get('map') is not None:
    val = int(float(data['map']))
    self.pids['010B'] = f"{val:02X}"

    self.pids['011F'] = f"{runtime >> 8:02X} {runtime & 0xFF:02X}"
```

### 3.2 Mode 02 (Freeze Frame) 지원

#### 개요
Mode 02는 DTC 발생 시점의 스냅샷 데이터를 제공합니다. 프론트엔드의 `FREEZE_DTC` PID를 지원하기 위해 필요합니다.

#### 구현 방법

**process_command 메서드에 추가**:
```python
elif cmd.startswith("02"):
    # 예: 02 02 00 (Mode 02, PID 02, Frame 00)
    if cmd == "020200" and self.dtcs:
        # 첫 번째 DTC를 Freeze Frame 코드로 응답
        code = self.dtcs[0]
        b1, b2 = self._dtc_to_hex(code)
        response = f"42 02 00 {b1:02X} {b2:02X}"
    else:
        response = "NO DATA"
```

**DTC 16진수 변환 로직 개선**:
```python
def _dtc_to_hex(self, code):
    """
    P0115 -> (01, 15) 형태의 16진수 변환
    예: P0115 = 0x0115, C0300 = 0x4300
    """
    prefix = code[0]
    # 16진수 문자열로 파싱 (P0115의 '0115' 부분)
    num_str = code[1:]
    
    # 16진수로 변환
    if all(c in '0123456789ABCDEF' for c in num_str.upper()):
        num = int(num_str, 16)
    else:
        num = int(num_str)  # 10진수 폴백
    
    # Prefix 매핑
    prefix_val = {'P': 0x0, 'C': 0x4, 'B': 0x8, 'U': 0xC}.get(prefix, 0)
    
    b1 = (prefix_val << 4) | (num >> 8 & 0x0F)
    b2 = num & 0xFF
    
    return b1, b2
```

### 3.3 CSV 매핑 구조 개선

#### 현재 문제점
- `update_from_csv_row`가 고정된 매핑만 사용
- 신규 PID의 CSV 컬럼을 유연하게 매핑할 수 없음

#### 개선 방안

**config.yml 매핑 예시**:
```yaml
vehicles:
  - id: "DEV_001"
    name: "YF Sonata"
    vin: "1HMXOBSGK693TJ7JR"
    csv_file: "scenarios/driving_normal.csv"
    mappings:
      "010C": { col: "rpm", formula: "x * 4" }
      "010D": { col: "speed", formula: "x" }
      "0105": { col: "temp", formula: "x + 40" }
      "0104": { col: "load", formula: "x * 2.55" }
      "0111": { col: "throttle", formula: "x * 2.55" }
      "010B": { col: "map", formula: "x" }
      "0110": { col: "maf", formula: "x * 100" }
      "010F": { col: "intake_temp", formula: "x + 40" }
    dtcs: ["P0115"]
```

**update_from_csv_row 개선**:
```python
def update_from_csv_row(self, row, mappings):
    """CSV 행을 PID별 매핑 규칙에 따라 업데이트"""
    for pid_code, map_info in mappings.items():
        col = map_info.get('col')
        if col not in row:
            continue
        
        try:
            val = float(row[col])
            formula = map_info.get('formula', 'x')
            result = eval(formula, {"x": val})
            
            # 변환된 값을 16진수로 PID에 저장
            if pid_code == '010C':  # RPM (2바이트)
                int_val = int(result)
                self.pids[pid_code] = f"{int_val >> 8:02X} {int_val & 0xFF:02X}"
            elif pid_code in ['010D', '0105', '0104', '0111', '010B', '010F']:  # 1바이트
                self.pids[pid_code] = f"{int(result):02X}"
            elif pid_code == '0110':  # MAF (2바이트)
                int_val = int(result)
                self.pids[pid_code] = f"{int_val >> 8:02X} {int_val & 0xFF:02X}"
        except Exception as e:
            logger.debug(f"Error mapping {pid_code}: {e}")
```

### 3.4 Engine Runtime 자동 증가

#### 구현 방법

**시작 감지**:
```python
def start(self):
    # ... 기존 코드 ...
    
    # Engine Runtime 타이머 시작
    self.engine_start_time = time.time()
    
    # 1초마다 Runtime 업데이트하는 스레드
    threading.Thread(target=self._update_runtime, daemon=True).start()
```

**자동 업데이트 워커**:
```python
def _update_runtime(self):
    """Engine Runtime을 1초마다 자동 업데이트"""
    while self.running:
        if self.engine_start_time:
            runtime = int(time.time() - self.engine_start_time)
            self.pids['011F'] = f"{runtime >> 8:02X} {runtime & 0xFF:02X}"
        time.sleep(1)
```

### 3.5 향상된 DTC 시나리오 지원

#### 동적 DTC 추가/제거

**config.yml에서 시나리오별 DTC 제어**:
```yaml
vehicles:
  - id: "DEV_FAULT"
    name: "Fault Scenario"
    csv_file: "scenarios/fault_scenario.csv"
    dtcs: ["P0115", "P0300"]  # 초기 DTC
    dtc_injection:
      - { time: 30, action: "add", code: "P0420" }    # 30초 후 P0420 추가
      - { time: 60, action: "remove", code: "P0115" } # 60초 후 P0115 제거
```

**구현**:
```python
def replay_worker(self):
    # ... 기존 CSV 재생 로직 ...
    
    # DTC 인젝션 처리
    start_time = time.time()
    dtc_injections = vehicle.get('dtc_injection', [])
    
    while self.running:
        elapsed = time.time() - start_time
        
        for injection in dtc_injections:
            if abs(elapsed - injection['time']) < 0.5:  # 0.5초 오차 허용
                if injection['action'] == 'add':
                    self.dtcs.append(injection['code'])
                    self.mil_on = True
                elif injection['action'] == 'remove':
                    if injection['code'] in self.dtcs:
                        self.dtcs.remove(injection['code'])
                    self.mil_on = len(self.dtcs) > 0
```
