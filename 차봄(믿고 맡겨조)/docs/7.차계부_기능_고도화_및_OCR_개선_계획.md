# 7. 차계부 기능 고도화 및 OCR 개선 계획

## 1. 개요
현재 영수증 스캔 및 정비 이력 조회 위주의 단순 기능을 사용자 편의성과 데이터 정확도를 높이기 위한 고도화 계획입니다.

## 2. 주요 개선 사항
### 2.1 기초 기능 보강
- **수동 입력 기능**: 영수증이 없는 경우에도 날짜, 항목, 금액, 메모 등을 직접 입력할 수 있는 기능 추가
- **수정 및 삭제 기능**: 등록된 정비/주유 이력을 수정하거나 잘못된 데이터를 삭제하는 관리 기능 (현재는 조회만 가능)
- **통합 필터링**: 
    - 차량별 (개별 차량 선택 및 전체 통합 보기 기능)
    - 항목별 (소모품군, 주유, 세차 등)
    - 기간별 (월별, 연도별 지출 합산 확인)

### 2.2 주유 및 충전 관리 확대
- **주유/충전 OCR 도입**: 정비 영수증 외에 주유 영수증 스캔 시 주유량(L), 단가, 유종 등을 자동으로 추출하는 기능
- **데이터 통합**: 정비 이력과 주유 기록을 한 화면에서 관리하여 차량 유지비 전체를 파악

### 2.3 UI/UX 및 시각화
- **지출 통계**: 리스트 중심에서 월별 지출 그래프, 항목별 지출 비중(파이 차트) 등 대시보드 형태의 UX 제공
- **검색 기능**: 내역 내 키워드 검색을 통한 과거 정비 기록 추적

## 3. OCR 기술 개선 및 최적화
### 3.1 AI 프롬프트 동적화 (Hard-coding 제거)
- **현상**: `ReceiptAnalyzerService` 내 AI 프롬프트에 소모품 항목들(`ENGINE_OIL`, `TIRES` 등)이 하드코딩되어 있어 DB 변경 시 대응 불가
- **개선**: DB의 `consumable_items` 마스터 데이터를 조회하여 프롬프트에 동적으로 주입하는 구조로 변경하여 확장성 확보

### 3.2 영수증 이미지 보관 및 검증
- **검토**: OCR 파싱 오류 시 원본 대조를 위해 영수증 이미지를 저장소(S3 등)에 보관하는 방안 검토
- **기대효과**: 데이터의 신뢰성 확보 및 사용자가 추후 증빙용으로 사진을 확인할 수 있는 경험 제공

## 4. 데이터 설계 방향
- **MaintenanceHistory 엔티티 확장**: 비정형 데이터(OCR 원본 제이슨 등)와 정형 데이터 간의 조화로운 저장 (현재 `ocr_data` 필드 적극 활용)
- **주유 데이터 연계**: 주유량, 연비 계산 등을 위한 추가 필드 설계 또는 통합 데이터 모델링

## 5. 단계별 추진 계획
1. **[1단계]** 수동 입력, 수정/삭제, 기초 필터링(차량별/월별) 및 리스트 UI 개선
2. **[2단계]** AI 프롬프트 동적화 및 주유/충전 OCR 파싱 로직 확장
3. **[3단계]** 지출 통계 대시보드 및 이미지 저장 기능 연동
