# YOLO 차량 외관 손상 탐지 모델 학습 가이드 (상세설명 포함)

이 문서는 차량 외관의 파손(Damage)을 탐지하고, 파손의 심각도(Severity)를 평가하는 모델을 학습하는 방법을 아주 상세하게 설명합니다.

## 1. 개요 (Overview)
이 스크립트는 차량의 **스크래치, 찌그러짐(Dent), 파손(Damage)** 등을 탐지하고, 이를 **3단계 심각도(Normal/Warning/Critical)**로 분류하는 기능을 포함합니다.

- **기본 모델**: YOLO11m (Medium)
- **학습 데이터**: `ai/data/yolo/exterior/data.yaml` (CarDD + 커스텀 데이터)
- **이미지 크기**: 640 (기본값) 또는 1280 (정밀 탐지 시 권장)

---

## 2. 실험 설계: 베이스라인 vs 도전자 (Baseline Definition) 🥊

엔진룸 모델과 마찬가지로, 기준점(Baseline)을 잡고 더 나은 모델(Challenger)을 만들어가는 방식으로 진행합니다.

| 역할 | 모델 정의 | 설명 | 목표 |
| :-- | :-- | :-- | :-- |
| **Baseline**<br>(기준점) | **YOLOv8n** (Nano) | 가장 가볍고 빠른 모델. 학습 속도가 매우 빠름. | "최소한 이 성능은 넘어야 한다"는 기준점 확인. |
| **Challenger 1**<br>(주력) | **YOLO11m** (Medium) | 현재 설정된 기본 모델. 성능과 속도의 균형. | 실사용 가능한 수준의 정확도 달성. |
| **Challenger 2**<br>(고성능) | **YOLO11x** (X-Large) | 현존 최강 성능 모델. 미세한 스크래치 탐지에 유리. | 아주 작은 파손까지 다 찾아내기 (VRAM 12GB+ 필요). |

### 2.1 Baseline 잡는 법 (모델 변경)
`train_exterior.py` 파일의 상단을 보면 `BASE_MODEL` 변수가 있습니다. 이 값을 바꿔서 실험을 진행합니다.

1.  `ai/scripts/vision/train_exterior.py` 파일을 엽니다.
2.  `BASE_MODEL` 값을 변경합니다.

```python
# [Configuration] RunPod Optimized Settings
# 1. Baseline 잡을 때 (가벼운 모델)
BASE_MODEL = "yolov8n.pt"  

# 2. 본 게임 (성능 모델)
BASE_MODEL = "yolo11m.pt" 
```

3.  학습을 돌려보고 나온 점수(mAP)를 기록합니다. 이 점수가 바로 **Baseline**입니다.

## 3. 학습 시작하기 (Training)

Baseline 점수를 확인했다면, 이제 본격적으로 성능을 올리는 학습을 시작합니다.

### 3.1 기본 학습 실행
터미널에서 아래 명령어를 실행합니다.

```bash
# 기본 설정 (YOLO11m, Epochs 10)
python ai/scripts/vision/train_exterior.py --mode train
```

### 3.2 고성능 학습 (추천)
성능을 높이려면 Epoch를 늘리고 이미지 크기를 키우세요. 특히 **스크래치**는 해상도가 높아야 잘 보입니다.

```bash
# Epochs 100, 이미지 크기 1280 (고해상도)
python ai/scripts/vision/train_exterior.py --mode train --epochs 100 --imgsz 1280
```

### 3.3 주요 설정값 (하이퍼파라미터) 설명
`train_exterior.py` 실행 시 조절 가능한 값들입니다.

| 옵션 (`--`) | 쉬운 설명 | 추천 값 |
| :--- | :--- | :--- |
| `epochs` | **공부 횟수**. 문제집을 처음부터 끝까지 몇 번 볼 거냐는 뜻입니다. | `100` ~ `300` (많을수록 똑똑해지지만 오래 걸림) |
| `batch` | **한 번에 공부할 양**. 컴퓨터 메모리가 부족하면 줄여야 합니다. | `16` (메모리 부족 시 `8`로 줄이세요) |
| `imgsz` | **이미지 크기**. 클수록 작은 스크래치를 잘 보지만 느립니다. | `640` (빠름), `1280` (정확함 - 스크래치용) |
| `device` | 그래픽카드 번호 | `0` (보통 0번 사용) |

## 4. 파손 심각도 (Severity) 분류 기준 ⚠️

이 모델은 단순히 "파손됐다"만 찾는 게 아니라, 파손된 부위가 **얼마나 심각한지(Severity)**를 3단계로 계산합니다.
스크립트(`train_exterior.py`) 내에 정의된 매핑 규칙은 다음과 같습니다.

### 🔴 Critical (심각 - 교체 필요)
**유리 파손, 라이트 깨짐** 등 즉시 수리가 필요한 상태입니다.
- **대상**:
  - `Front-Windscreen-Damage` (앞유리 파손)
  - `Rear-windscreen-Damage` (뒷유리 파손)
  - `Headlight-Damage` (헤드램프 파손)
  - `Taillight-Damage` (테일램프 파손)
  - `Signlight-Damage` (방향지시등 파손)

### 🟠 Warning (주의 - 판금/도색 필요)
**찌그러짐(Dent)**이 심한 상태입니다.
- **대상**:
  - `bonnet-dent`, `doorouter-dent`, `fender-dent`
  - `trunk-dent` (rear-bumper-dent 등)
  - `roof-dent`, `pillar-dent`
  - 그 외 주요 패널의 찌그러짐

### 🟢 Normal/Minor (경미 - 광택/컴파운드 가능)
**단순 스크래치(Scratch)**나 도장 벗겨짐 정도입니다.
- **대상**:
  - `doorouter-scratch`, `front-bumper-scratch`, `rear-bumper-scratch` (스크래치류)
  - `paint-chip`, `paint-trace` (도장 까짐)
  - `Sidemirror-Damage` (사이드미러 손상 - *현재 설정상 Minor로 분류됨*)

## 5. 성능 평가 (Evaluation)

학습이 끝난 모델이 파손도 잘 찾고, 심각도(Severity)도 잘 맞추는지 확인합니다.

```bash
python ai/scripts/vision/train_exterior.py --mode test
```

### 결과 확인
1.  **mAP (탐지 정확도)**: 파손 위치를 얼마나 잘 찾았는지.
2.  **Severity Accuracy (심각도 정확도)**: 찾은 파손을 '심각/주의/경미'로 잘 구분했는지. (스크립트 마지막에 `📊 Severity Accuracy Results`로 출력됩니다.)

## 7. 자주 묻는 질문 (FAQ)

**Q. 스크래치가 잘 안 잡혀요.**
A. **조명(빛)**이 중요합니다! 스크래치는 빛이 반사될 때 잘 보입니다. 그리고 **이미지 크기(`imgsz`)를 1280**으로 키워서 학습시켜 보세요.

**Q. 찌그러짐(Dent)이 잘 안 잡혀요.**
A. 찌그러짐은 정면보다 **측면**에서 찍었을 때 그림자가 생겨서 더 잘 보입니다. 데이터 수집 시 다양한 각도에서 찍어주세요.

**Q. 학습 시간이 너무 오래 걸려요.**
A. **`imgsz`를 640**으로 줄이거나, **`batch` 사이즈를 키워보세요(메모리가 허락하는 한)**. 또는 더 가벼운 모델(`YOLO11s`, `YOLOv8n`)을 고려해보세요.

## 8. 데이터 부족(Data Scarcity) 해결 전략 🚨

현재 데이터 분포를 보면 **스크래치(Scratch)나 칠 까짐(Paint-chip)** 데이터가 매우 부족합니다(10장 미만). 이대로 학습하면 AI는 이 결함을 아예 못 찾을 수 있습니다. 다음 전략을 꼭 사용하세요.

### 8.1 생성형 AI 활용 (Generative AI) 🤖
"데이터가 없으면 만들면 됩니다." 요즘 AI들은 진짜 같은 가짜 이미지를 아주 잘 만듭니다.
*   **방법**: Stable Diffusion, Midjourney, Photoshop Generative Fill 등을 사용합니다.
*   **프롬프트(명령어) 예시**:
    *   `"Close-up photo of a white car door with a deep scratch, realistic, 4k"`
    *   `"Silver car fender with paint chip damage, detailed"`
*   **활용**: 이렇게 만든 이미지를 50~100장 정도 확보해서 학습 데이터에 섞어주세요.

### 8.2 강력한 증강 기법 (Strong Augmentation) 💪
있는 데이터를 뻥튀기하는 기술입니다.
*   **Copy-Paste (오려 붙이기)**:
    *   현재 있는 6장의 스크래치 사진에서 **스크래치 부분만 오려냅니다.**
    *   그리고 정상적인 범퍼나 문짝 사진(1500장) 위에 랜덤하게 **붙여넣습니다.**
    *   이러면 순식간에 수천 장의 스크래치 데이터를 만들 수 있습니다. (가장 효과적인 방법 중 하나입니다!)

### 8.3 외부 데이터 수집 (크롤링) 🌍
직접 찍는 게 힘들다면 인터넷 세상을 뒤져봅시다.
*   **YouTube**: "Car scratch repair", "Car detailing before after" 영상을 검색하세요. 영상 속 스크래치 장면을 캡처하면 아주 좋은 데이터가 됩니다.
*   **Google Images**: "Car door scratch", "Bumper paint chip" 등으로 검색해서 줍줍하세요.

> **결론**: 데이터가 10장 미만이면 학습이 불가능합니다. 위 3가지 방법 중 하나라도 써서 최소 **50장** 이상은 만들어주세요!

## 9. AI 학습용 데이터(라벨링) 만들기 가이드 🏷️

"사진은 찍었는데, 이걸 컴퓨터에게 어떻게 가르치나요?" -> **라벨링(Labeling)**을 해야 합니다.
라벨링이란 사진 속의 파손 부위에 **직접 네모 박스를 치고 "이게 스크래치야"라고 이름표를 달아주는 작업**입니다.

### 9.1 라벨링 도구 추천 (Tools)

#### 1) Roboflow (추천! ⭐) - 웹 기반
설치가 필요 없고 웹사이트에서 바로 할 수 있어 가장 쉽습니다.
- **장점**: 어디서든 가능, 자동 저장, 팀원과 공유 편함.
- **주소**: [https://roboflow.com/](https://roboflow.com/)
- **방법**: 회원가입 -> 프로젝트 생성 -> 이미지 업로드 -> 마우스로 박스 그리기 -> **Export (YOLO v8 Format)**

#### 2) LabelImg - 설치형
인터넷 없이 내 컴퓨터에서 하고 싶을 때 씁니다.
- **설치**: 파이썬이 깔려있다면 `pip install labelimg` 입력 후 실행.
- **단점**: 단축키를 익혀야 빠름. (W: 박스 그리기, A/D: 이전/다음 사진)
- **주의**: 저장 포맷을 반드시 **YOLO**로 설정하세요! (기본값은 PascalVOC일 수 있음)

### 9.2 라벨링 잘하는 법 (Best Practices)
AI의 성능은 라벨링 품질이 결정합니다. 대충 하면 AI도 대충 배웁니다.

1.  **박스는 타이트하게!** 📦
    *   스크래치 주변에 여백을 너무 많이 주지 마세요. 스크래치에 딱 맞게 박스를 쳐야 AI가 정확히 배웁니다.
2.  **일관성 있게!**
    *   "이 정도 흠집도 잡을까?" 고민된다면 팀원들과 기준을 정하세요. 누구는 잡고 누구는 안 잡으면 AI가 헷갈려합니다.
3.  **이름 확인!**
    *   `scratch`, `dent` 처럼 영어 이름을 정확히 써주세요. 오타가 나면 다른 클래스로 인식합니다.

### 9.3 데이터셋 내보내기 (Export)
라벨링이 끝나면 데이터를 **YOLOv8** 형식으로 내보내야 합니다.
- **Roboflow**: `Export Dataset` 버튼 클릭 -> Format: `YOLOv8` 선택 -> `Download zip`
- 압축을 풀면 `images`와 `labels` 폴더가 나옵니다. 이 폴더를 `ai/data/yolo/exterior/` 경로에 넣어주면 준비 끝!

## 10. 심화: 전처리 및 증강 전략 (Preprocessing Strategy) 🖼️

"같은 사진이라도 어떻게 보여주느냐에 따라 AI 성능이 달라집니다."
외관(Exterior) 학습 시에는 **스크래치가 잘 보이게** 만드는 것이 핵심입니다.

### 10.1 외관(Exterior) 핵심 전략
야외 촬영이 많고, 미세한 흠집을 잡아야 하므로 다음 설정을 권장합니다.

| 설정 항목 | 추천 값 | 이유 |
| :-- | :-- | :-- |
| **해상도 (imgsz)** | **1280** (고해상도) | **가장 중요!** 스크래치는 얇은 선입니다. 640으로 줄이면 뭉개져서 안 보입니다. |
| **Mosaic (모자이크)** | **1.0** (켜기) | 작은 파손 부위를 학습하는 데 아주 효과적입니다. (YOLO 기본값) |
| **HSV (색상/밝기)** | **강하게** | 차는 낮/밤/흐린 날 등 조명 변화가 심합니다. 밝기 변화를 주어 다양한 환경에 적응시킵니다. |
| **Flip (좌우 반전)** | **0.5** (켜기) | 차의 좌우는 대칭이므로, 좌우 반전은 데이터 2배 효과를 냅니다. (상하 반전은 끄세요!) |

### 10.2 코드 적용 예시
`train_exterior.py`에서 `model.train()` 함수 안에 다음 옵션을 추가하면 적용됩니다.

    mosaic=1.0        # 7. 모자이크 증강 켜기 (작은 객체 탐지 유리)
)
```

## 11. [부록] 모델 출력 데이터 예시 (JSON Structure)

### 11.1 차량 외관 분석 (SCENE_EXTERIOR)

```jsonc
{
  "status": "WARNING", // NORMAL, WARNING, CRITICAL, ERROR
  "analysis_type": "SCENE_EXTERIOR",
  "category": "EXTERIOR",
  "data": {
    "damage_found": true, // 파손 발견 여부
    "detections": [
      {
        "part": "Front Bumper", // 파손 부위
        "damage_type": "Scratch", // 파손 종류
        "confidence": 0.88, // 정확도
        "bbox": [100, 500, 200, 100] // [x1, y1, x2, y2]
      }
    ]
  }
}
```

#### 📋 탐지 항목 정의
*   **감지 부위 (Part)**:
    *   `Bonnet`, `Door`, `Fender`, `Front Bumper`, `Rear Bumper`
    *   `Quarter Panel`, `Roof`, `Pillar`, `Running Board`, `Body Panel`
    *   `Front Windshield`, `Rear Windshield`, `Headlight`, `Taillight`, `Sidemirror`
*   **파손 종류 (Damage Type)**:
