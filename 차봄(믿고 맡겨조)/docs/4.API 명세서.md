# API 상세 명세서 (v2.0)

> **문서 개요**: 본 문서는 시스템 간 통신 규격을 정의하며, 크게 **Client(App) ↔ Backend(Java)** 와 **Backend(Java) ↔ AI Server(Python)** 두 가지 파트로 나뉩니다.
> **RabbitMQ**는 내부 인프라 통신용이므로 본 REST API 명세에는 포함되지 않습니다.

# Part 1. Frontend ↔ Backend API
> **Mobile App**과 **Java Backend** 간의 통신 규격입니다.
> - **Base URL**: `https://api.car-sentry.com/api/v1`
> - **Global Prefix**: 모든 API 엔드포인트 앞에는 자동으로 `/api/v1`이 붙습니다. (아래 경로는 이 접두어 이후의 상대 경로입니다.)
> - **Auth**: `Authorization: Bearer {jwt_token}`

### ?? 공통 응답 규격 (Common Response)
모든 API 응답은 `ApiResponse` 객체로 래핑되어 전달됩니다.
```json
{
  "success": true,
  "data": { ... },
  "error": null
}
```
- **success**: 요청 처리 성공 여부 (boolean)
- **data**: 실제 결과 데이터 (성공 시에만 존재, 실패 시 null)
- **error**: 에러 정보 (실패 시에만 존재, 성공 시 null)
  - `code`: 에러 코드
  - `message`: 상세 에러 메시지

## 1. 사용자 및 인증 (Auth & Users)
> Base Path: `/auth`

### 1.1 회원가입 (Signup)
- **POST `/auth/signup` (BE-AU-001)**
- **Request Body**:
  ```json
  {
    "email": "user@example.com",
    "password": "password123",
    "nickname": "carlover"
  }
  ```
- **Response**: `ApiResponse<UserResponse>`
  ```json
  {
    "success": true,
    "data": {
      "userId": "UUID",
      "email": "user@example.com",
      "nickname": "carlover",
      "profileImageBase64": "string (null if empty)"
    },
    "error": null
  }
  ```

### 1.2 로그인 및 토큰 (Login & Token)
- **POST `/auth/login` (BE-AU-002)**: JWT 토큰 발급
- **Request Body**:
  ```json
  {
    "email": "user@example.com",
    "password": "password123"
  }
  ```
- **Response**: `ApiResponse<TokenResponse>`
  ```json
  {
    "success": true,
    "data": {
      "accessToken": "ey...",
      "refreshToken": "ey..."
    },
    "error": null
  }
  ```

- **POST `/auth/refresh`**: 토큰 갱신
- **Request Body**: `{ "refreshToken": "string" }`
- **Response**: `ApiResponse<TokenResponse>`

### 1.3 사용자 프로필 (User Profile)
- **GET `/auth/me` (BE-AU-003)**: 내 프로필 조회
- **Response**: `ApiResponse<UserResponse>`

- **PATCH `/auth/me` (BE-AU-004)**: 프로필 수정
- **Request Body**: `{ "nickname": "string", "memo": "string" }`

- **POST `/auth/me/image`**: 프로필 이미지 업로드

- **PATCH `/auth/fcm-token`**: FCM 토큰 전용 갱신

- **DELETE `/auth/me` (BE-AU-007)**: 회원 탈퇴 (Soft Delete)

## 4. 텔레메트리 및 운전 분석 (Telemetry)
> Base Path: `/telemetry`

### 4.1 OBD 데이터 수집
- **POST `/telemetry/batch` (BE-TD-002)**: OBD 로그 배치 업로드 (3분 주기)
- **Request Body**: `List<ObdLogDto>` (timestamp, vehicleId, rpm, speed 등 포함)
- **Response**: `ApiResponse<Void>`

- **GET `/telemetry/status/{vehicleId}` (BE-TD-006)**: 차량 실시간 연결 상태 조회
- **Response**: `ApiResponse<ConnectionStatusDto>` (`connected`, `lastDataTime`, `statusMessage`)

- **POST `/telemetry/status/disconnect`**: 차량 수동 연결 해제 (세션 강제 종료)
- **Request Body**: `{ "vehicleId": "UUID" }`

### 4.2 OBD 장치 및 차량 특정 (v1.7)
> Base Path: `/obd` (신규)

- **GET `/obd/devices`**: 현재 사용자 OBD 장치 목록 (앱 기동 시 1회·캐시 갱신용)
- **Response**: `ApiResponse<List<ObdDeviceDto>>`
  ```json
  { "success": true, "data": [
    { "id": "UUID", "deviceId": "BLE_UUID_or_address", "deviceType": "ble", "name": "내 OBD" }
  ]}
  ```

- **POST `/obd/devices`**: OBD 장치 등록
- **Request Body**: `{ "deviceId": "string", "deviceType": "ble"|"classic", "name": "string (optional)" }`
- **Response**: `ApiResponse<ObdDeviceDto>`

- **PUT `/obd/devices/{deviceId}/connect`**: 연결 시 히스토리 기록 (마지막 연결 차량·CALID/CVN 갱신)
- **Request Body**: `{ "vehicleId": "UUID", "calid": "string (optional)", "cvn": "string (optional)" }`
- **Response**: `ApiResponse<Void>`

- **POST `/obd/resolve-vehicle`**: 차량 특정 (VIN 실패 시 CALID/CVN·마지막 연결·대표 차량 순)
- **Request Body**: `{ "deviceId": "string", "vin": "string (optional)", "calid": "string (optional)", "cvn": "string (optional)" }`
- **Response**: `ApiResponse<{ "vehicleId": "UUID" }>`
- **로직 순서**: 1) vin 있으면 user 소유 vehicles에서 VIN 매칭 → vehicleId 반환. 2) 없으면 calid/cvn으로 obd_device_vehicle_history 매칭. 3) 없으면 해당 device의 last_connected_at 최대 vehicle. 4) 없으면 user의 is_primary=true 차량.

- **Request Body**: `{ "tripId": "UUID" }`
- **Response**: `ApiResponse<TripSummary>`
  ```json
  {
    "success": true,
    "data": {
      "tripId": "UUID",
      "vehicleId": "UUID",
      "startTime": "2024-01-17T14:30:00",
      "endTime": "2024-01-17T15:00:00",
      "distance": 15.5,
      "driveScore": 95,
      "averageSpeed": 45.0,
  ```

- **GET `/trips` (BE-TD-005)**: 내 주행 이력 목록 조회 (파라미터: vehicleId)

### 6.4 DTC 보고 (배치) [NEW]

- **POST `/ai/dtc/batch` (BE-AI-002)**  
  - **Description**: Mode 03(Stored DTCs) + Mode 02(Freeze Frame)을 한 번에 배치로 보고합니다.  
  - **사용 주체**: 모바일 앱의 OBD 서비스 (`sendDtcBatchReport`)가 사용하며, **단일 코드 보고(`POST /ai/dtc`)는 하위 호환/테스트용**으로만 사용됩니다.

- **Request Body**: `DtcBatchRequest`
  ```json
  {
    "vehicleId": "UUID",
    "dtcs": [
      { "code": "P0420", "type": "STORED", "status": "ACTIVE" },
      { "code": "P0171", "type": "STORED", "status": "ACTIVE" },
      { "code": "P0300", "type": "FREEZE_FRAME", "status": "ACTIVE" }
    ],
    "freezeFrame": {
      "rpm": 3200.0,
      "speed": 80.0,
      "coolantTemp": 92.0,
      "engineLoad": 45.0,
      "ambientTemp": 27.0,
      "fuelPressure": 280.0,
      "pidsSnapshot": "{\"rpm\":3200,\"speed\":80,...}"
    }
  }
  ```

  - `vehicleId`: 차량 ID (UUID)
  - `dtcs`: 이번 보고에 포함된 DTC 코드 목록
    - `code`: DTC 코드 (예: `P0420`)
    - `type`: `STORED` / `PENDING` / `PERMANENT` / `FREEZE_FRAME` 중 하나  
      - `FREEZE_FRAME`: Mode 02에서 캡처된 “대표 DTC”에 사용
    - `status`: `ACTIVE` / `RESOLVED` / `CLEARED` / `PENDING` / `STORED`
  - `freezeFrame`: Mode 02 Freeze Frame 데이터 (있을 때만 전송)
    - `rpm`, `speed`, `coolantTemp`, `engineLoad`, `ambientTemp`, `fuelPressure`: 주요 센서 값
    - `pidsSnapshot`: 기타 PID를 JSON 문자열로 직렬화한 스냅샷

- **Response**: `ApiResponse<Void>`
  ```json
  {
    "success": true,
    "data": null,
    "error": null
  }
  ```

> [!NOTE]
> - 프론트는 ECU의 01 01(DTC Status)을 주기적으로 폴링하고,  
>   **“DTC 개수 0 → N(>0)” 변화가 감지되는 시점에만** Mode 03/02를 조회하여 `POST /ai/dtc/batch`를 호출합니다.  
> - 백엔드는 배치 단위로 `dtc_history`(코드 이력)와 `dtc_freeze_frames`(Freeze Frame 스냅샷)에 데이터를 저장한 뒤,  
>   필요 시 FCM 알림 및 AI 진단 파이프라인을 트리거합니다.

### 1. Vision Analysis (YOLOv8)
- **POST `/predict/visual`**
    - **Description**: 차량 외관 이미지를 분석하여 손상 부위 탐지.
    - **Request (Multipart)**:
        - `file`: 이미지 파일 (Binary)
    - **Response (JSON)**:
        ```json
        //1. 엔진룸
        ```

### 3. Anomaly Detection (Engine Hybrid + Rule Domains)

- 3.1 **POST `/predict/anomaly`**
  - **Description**: 최근 일정 시간(기본 60초 윈도우) 동안 누적된 시계열 OBD 데이터 배열을 입력으로 받아,
    - Core(엔진 ML: **Stat One-Class + LSTM-AE Hybrid**) +
    - 도메인별 Rule 기반 이상 여부(electrical/brake/tire/idle)
    를 분석합니다.
  - **Route Path Note**: AI Server 라우터 기준 실제 호출 경로는 `POST /api/v1/predict/anomaly` 입니다.
  - **Execution Note**:
    - 현재 서비스 구현은 윈도우를 순차 처리합니다.
    - 엔진 내부는 데이터 품질 게이팅(AE_ONLY / IF_ONLY / BOTH)에 따라 단일/앙상블 점수를 계산합니다.
  - **Response 기본 정책**: `summary`
  - **응답 구조 원칙**:
    - `extensions` 래퍼는 사용하지 않습니다.
    - `domains + events[]` 중심으로 반환합니다.
    - 이상 이벤트 발생 시 `events[].feature`를 반드시 포함합니다.

#### Engine Core Features (Core7)
엔진 ML은 아래 7개 feature를 **표준 스키마(Core7)** 로 사용합니다.

1. engine_coolant_temp_c   (냉각수 온도)

> 누락 시 엔진 ML은 **Schema Align + Fallback Policy**를 따릅니다.

- 3.2 **Domain Result Schema**

```json
{
  "domain": "string",
  "status": "PROCESSED | SKIPPED | UNSUPPORTED | ERROR",
  "score": "number | null",
  "threshold": "number | null",
  "is_anomaly": "boolean",
  "top_signals": [
    {
      "feature": "string",
      "contribution": "number(0~1)"
    }
  ]
}
```

- `domain (string)`: 예) `engine` (코어 결과 도메인)
- `status (string)`: `PROCESSED | SKIPPED | UNSUPPORTED | ERROR`
  - `PROCESSED`: 정상 분석 완료
  - `SKIPPED`: 조건 불충족으로 미수행 (예: `mode=DRIVING`에서 `idle`)
  - `UNSUPPORTED`: 차량/데이터 상 지원하지 않는 도메인
  - `ERROR`: 분석 중 오류 발생
- `score (float|null)`: 이상 점수(0~1). 값이 클수록 정상 패턴과의 차이가 큼
- `threshold (float|null)`: 이상 판단 기준값
- `is_anomaly (bool)`: `score >= threshold`인 경우 `true`
- `top_signals (array<object>, optional)`:
  - 이상판단에 크게 기여한 센서(feature) Top-k
  - `feature (string)`: 센서(feature) 이름
  - `contribution (float)`: 상대 기여 비중(0~1)
  - **최대 `top_k`개 반환** (항상 `top_k`개 고정 아님)
  - 주로 ML 도메인(예: engine)에서 제공, Rule 도메인은 생략 가능 또는 빈 배열 가능

---

- 3.3 **Event Schema**

```json
{
  "type": "string",
  "domain": "string",
  "feature": "string",
  "value": "number | string | null",
  "threshold": "number | null",
  "window_index": "number | null",
  "severity": "INFO | WARNING | CRITICAL",
  "message": "string"
}
```

- 이상 이벤트는 `feature` 필드 포함 필수
- 백엔드 전달/알림/로그/후속 리포트 생성은 `events[]`를 표준으로 사용
- severity 해석:
  - `INFO`: 참고/품질 안내 이벤트 (정상 확정 의미 아님)
  - `WARNING`: 주의 필요 이벤트 (점검 권장)
  - `CRITICAL`: 즉시 대응 필요 이벤트 (우선 조치)

---

- 3.4 **Raw Response Policy**

- `options.return = "summary"`: `window_results`는 빈 배열 또는 생략
- `options.return = "raw"`: `window_results` 포함 (디버깅/분석용)

---

- 3.5 **Compatibility & Deprecation**

- **Deprecated (응답)**: `core`, `extensions`, `details.rules` 중심 중첩 구조
- **Deprecated (입력)**: `options.extensions`
- **권장 표준 (응답)**: `is_anomaly + anomaly_score + domains + events[]`

### 6. AI 진단 통합 (LLM Generation)
- **POST `/generate/diagnosis`** 또는 **POST `/connect/predict/comprehensive`**
    - **Description**: 수집된 모든 멀티모달 데이터와 RAG 검색 결과를 종합하여, LLM 기반의 최종 진단 리포트를 생성합니다.
    
    > **Java Backend → AI Server Payload Mapping**  
    > Java에서 전송하는 `AiUnifiedRequestDto`는 다음 필드를 포함합니다:
    > - `visual_analysis`, `audio_analysis`, `anomaly_analysis` (각 AI 모델 분석 결과)
    > - `knowledge_data` (RAG 검색 문단 리스트)
    > - `vehicle_info`, `consumables_status` (차량/소모품 정보)
    > - `dtc_info` (DTC 모드에서만 제공, dtc_list 포함)
    > - `conversation_history` (reply 모드 시)
    >
    > **AI Server 어댑터 역할**  
    > Python AI Server는 다음과 같이 변환하여 OpenAI에 전달합니다:
    > - `diagnosis_context` ← 내부 계산 (overall_confidence, data_sources, is_confirmed_source)
    > - `analysis_results.sound` ← `audio_analysis`
    > - `analysis_results.image` ← `visual_analysis`
    > - `analysis_results.lstm` ← `anomaly_analysis`
    > - `rag_context` ← `knowledge_data`
    > - `dtc_info` ← `dtc_info` (그대로 전달)
    > - `vehicle_info`, `consumables_status` ← 그대로 전달
    
    - **Request (JSON)** - 프롬프트 입력 구조 (AI Server가 OpenAI에 전달하는 형식):
        ```json

            "vehicle_info": {
                "model": "Avante",
                "year": 2012,
                "fuel_type": "GASOLINE",
                "mileage": 150000
            },
            "dtc_info": { // DTC 모드에서만 제공

                    {

                "sound": {
                    "detected": true,

                }
            },
            "rag_context": [ // RAG 검색 결과 (2010~2013 매뉴얼)
                "[DTC Definition (En)] Catalyst System Efficiency Below Threshold...",
                "[DTC Definition (Ko)] 촉매 시스템 효율 저하...",
                "냉각수 온도가 110도 이상일 경우 과열로 간주하며...",
        ```
