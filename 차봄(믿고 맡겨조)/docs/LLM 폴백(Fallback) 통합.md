# 감사 보고서: LLM 폴백(Fallback) 통합

이 보고서는 AI 분석 파이프라인 전반의 LLM 폴백 메커니즘 감사 결과를 담고 있습니다. 주요 목표는 연결성, 트리거 조건, 그리고 폴백 로직의 견고함을 검증하는 것이었습니다.

## 분석 결과 요약

코드베이스는 진단 정확도와 신뢰성을 극대화하기 위해 설계된 정교한 다계층 폴백 전략을 구현하고 있습니다. 기존 GPT-5에서 **GPT-4o**로의 전환(강제 고정)을 통해 JSON 기반 응답 및 시각 분석의 안정성이 크게 향상되었습니다.

---

## 1. 다계층 폴백 아키텍처

### 레이어 1: 글로벌 라우터 레벨
- **위치**: `visual_service.py`
- **메커니즘**: `RouterService`가 장면을 분류할 때 **신뢰도가 0.4 미만**인 경우, 시스템은 즉시 "범용 시각 폴백(General Vision Fallback)"을 시작합니다.
- **작업**: GPT-4o가 이미지를 분석하여 도메인(엔진, 외관 등)을 결정하고 1차 진단을 생성합니다.

### 레이어 2: 도메인 특화 결정 레이어 (Gates)
- **위치**: `dashboard_service.py`, `exterior_service.py`, `tire_service.py`
- **메커니즘**: 표준화된 게이트 시스템 (0-4).
  - **게이트 0 (MISS)**: YOLO가 아무것도 찾지 못했을 때, 상위 레벨의 "세이프티 넷(Safety Net)"이 LLM을 트리거합니다.
  - **게이트 3/4 (Uncertain)**: YOLO 신뢰도가 경계선에 있을 때, LLM을 사용하여 "검증" 또는 "라벨 재지정"을 수행합니다.
- **결과**: 기본 YOLO 모델이 실패하더라도 사용자는 빈 오류 대신 지능적인 분석 결과를 받을 수 있습니다.

### 레이어 3: 이상 탐지 이중 점검 (Dual-Check)
- **위치**: `engine_anomaly_service.py`
- **메커니즘**: 
  - PatchCore가 이상을 감지하면, GPT-4o가 원본 크롭 이미지와 히트맵 오버레이를 사용하여 이를 교차 검증합니다.
  - **오탐 억제(False Positive Suppression)**: PatchCore의 플래그에도 불구하고 LLM이 "정상(NORMAL)"으로 결론 내리면 LLM의 의미론적 판단을 우선시합니다.

### 레이어 4: 오디오 에스컬레이션
- **위치**: `audio_service.py`
- **메커니즘**: 시각 분석과 유사하게, AST 모델의 신뢰도가 낮거나 라벨이 불확실한 경우 시스템은 **GPT-4o-audio-preview**로 전환합니다.
- **전처리**: LLM으로 전송하기 전에 노이즈 및 음성 제거 전처리 과정이 포함됩니다.

## 2. 주요 구현 세부 사항

| 컴포넌트 | 기본 모델 | 폴백 트리거 조건 | 폴백 모델 | 역할 |
| :--- | :--- | :--- | :--- | :--- |
| **라우터** | EfficientNet | 신뢰도 < 0.4 | GPT-4o | 도메인 식별 |
| **엔진** | YOLO11m + PatchCore | 이상 점수 > 임계값 | GPT-4o | 검증 |
| **외관** | YOLOv11m | 게이트 0/3/4 | GPT-4o | 정밀 분석(Discovery) |
| **계기판** | YOLOv8 | 게이트 0/3/4 | GPT-4o | 정밀 분석(Discovery) |
| **타이어** | YOLO (분류) | 신뢰도 낮음 | GPT-4o | 오라클(Oracle) |
| **오디오** | AST | 결정 상태 == UNCERTAIN | GPT-4o Audio | 검증 |

---

## 3. 액티브 러닝(Active Learning) 통합
- **오라클 라벨링**: 모든 폴백 이벤트 또는 저신뢰 탐지(시각 < 0.85, 오디오 < 0.85)는 `generate_training_labels` 또는 `generate_audio_labels`를 트리거합니다.
- **S3 영구 저장**: 데이터는 `dataset/[도메인]/llm_confirmed/`에 저장되어 향후 재학습을 위한 고가치 정답지(Ground Truth)로 활용됩니다.
- **매니페스트 트래킹**: 학습 파이프라인 구축을 위해 `manifest_service`에 항목이 자동으로 기록됩니다.

## 4. 검증 결론
LLM 폴백 통합은 **완벽하게 연결되어 있으며 매우 견고함**을 확인했습니다. 시스템은 다음 상황들을 성공적으로 처리합니다:
1. **모델 실패**: YOLO/AST 오류 또는 빈 결과값 발생 시.
2. **모호한 데이터**: 저신뢰 탐지 결과 발생 시.
3. **도메인 불일치**: 라우터 오류 발생 시 (예: 엔진룸을 계기판 분석으로 리다이렉션).

> [!IMPORTANT]
> 복잡한 JSON 형식 작업 시 GPT-5에서 관찰된 안정성 문제로 인해, 모든 핵심 LLM 경로는 **GPT-4o** 또는 **GPT-4o-audio-preview**로 고정되었습니다.
